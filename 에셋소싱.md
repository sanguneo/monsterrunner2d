> 3. **인게임 엔티티**(플레이어·몬스터·보스·장애물·발사체·픽업·환경)는 **2D 프리미티브(캔버스 도형)로 선(先)동작 → 2D 스프라이트 시트로 후(後)교체**.
> **원칙**: 시각·로직(view-logic) 분리 — 로직은 엔티티 `id`만 참조, 스프라이트는 매니페스트 교체만으로 반영.

## 0. 공통 형식 / 기술 스펙

### 0.1 디바이스 / 성능 예산
- 타깃: 중급폰 이상(Galaxy S / iPhone 12+ 급), **가로(Landscape)** 모바일 브라우저.
- 목표 프레임레이트: 60fps(최소 30fps 하한).
- Canvas 2D 예산: 동시 `drawImage`/도형 호출 ~수백 회 이내. 오프스크린 캔버스 캐시·스프라이트 아틀라스로 드로우 콜 절감.
- 텍스처(스프라이트 아틀라스) 총량: 초기 로드 **≤ 2MB** 목표(월드별 지연 로드).

### 0.2 2D 스프라이트 (인게임 엔티티 — 신규 제작)
| 항목 | 스펙 |
|------|------|
| 포맷 | **PNG(알파) 또는 WebP(알파)**, 스프라이트 시트(아틀라스 + JSON 프레임 맵) 권장 |
| 해상도 | 논리 좌표 960×540 기준. 캐릭터 ~96×96, 몬스터 64~96px, 보스 160~256px, 장애물/픽업 48~96px |
| 애니메이션 | 프레임 시퀀스(달리기/점프/슬라이드/피격/공격/사망) — 8~12fps, 프레임 4~8장 |
| 방향 | **우측 진행 기준(오른쪽을 봄)**. 좌향 필요 시 `ctx` 수평 반전 |
| 피벗 | 하단 중앙(발밑=줄 기준선) 정렬 — 그림자·정렬 계산 단순화 |

### 0.3 사운드 (★ 인터페이스만 구현, 에셋 미포함)
| 항목 | 스펙 |
|------|------|
| 포맷 | **.ogg(주) + .mp3(폴백)**, 44.1kHz |
| SFX | 짧게(≤2s), −3dBFS 이하 피크. BGM은 루프 마킹 |
| 볼륨 | `SoundManager.setVolume`로 채널별 조절, 기본 SFX 0.8 / BGM 0.5 |

### 0.4 라이선스 정책
- 상용/배포 가능 라이선스만 사용(CC0 / CC-BY / 직접 제작 / 구매 에셋).
- 출처·라이선스·작가를 §12 부록(또는 `에셋출처.csv`)에 기록. CC-BY는 크레딧 표기 의무.
- 어린이 대상 콘텐츠 — 과도한 공포·폭력·선혈 표현 지양(전 월드 '귀엽게 으스스한' 톤, 좀비·드라큘라·해골도 카툰풍).

### 0.5 view-logic 교체 규약
- 각 엔티티는 `draw(ctx, x, y, ...)` 메서드로 **지금은 프리미티브 도형**을 그린다.
- 스프라이트 도입 시 `draw()` 내부만 `ctx.drawImage(atlas, frame…)`로 교체. 로직·좌표·충돌은 무변경.
- 스프라이트 매니페스트(`id → 아틀라스/프레임`)로 경로 해석 → 파일 교체 시 코드 무변경.

### 0.6 ✅ 기존 2D WebP 에셋 — 그대로 재사용 (신규 제작 불필요, 총 35개)
> 이미 제작·배선 완료. 2D 캔버스 전환과 무관하게 DOM 오버레이(HUD/Screens)에서 그대로 동작한다.

- **UI 아이콘 22종** — `public/assets/icons/*.webp` (`ui/icons.ts` `uiIcon()`/`worldThemeIcon()`):
  `coin, gem, heart, heal, dash, blast, rapid, star, trophy, crown, skull, ghost, party, distance, sparkle, pause` + 월드 테마 6종 `theme_school/zombie/lab/sea/dracula/skull`.
- **이미지 13종** — `public/assets/images/*.webp`:
  스테이지/월드 6 (`world_1_school … world_6_skull`), 타이틀 배경 1 (`title_bg`), 보상 장비 6 (`reward_ghost_cape, reward_zombie_hat, reward_lightning_helmet, reward_seawitch_crown, reward_dracula_cape, reward_skull_crown`).
- 캔버스 픽업(동전/보석/회복)에도 `coin/gem/heal.webp`를 `drawImage`로 재활용 가능(선택).

### 3.3 보스 공통 연출 에셋
- 경직(약점): 빨강 윤곽 발광(아웃라인) + "약점!" 팝업 + 데미지 숫자 확대·노란색.
- 페이즈 전환: 화면 플래시, 체력바 색(초록→주황→빨강) + 페이즈 눈금. 최종 페이즈 진입 시 배경 암전(`bgDark`).
- 등장: 오른쪽에서 진입 + 이름 배너.
- 임시는 전부 캔버스 도형/CSS로 구현, 최종은 이펙트 스프라이트/파티클로 교체.

## 4. 장애물 (월드 테마 색 적용)

> **줄 블로커 2종(BLOCK/MOVER)** 공통 도형 + **월드별 색상 테마**(`WorldTheme.obsLow/obsHigh/obsBlock` → BLOCK/MOVER 색 스킴으로 정리).
> 최종 교체 시 색뿐 아니라 월드별 소품 스프라이트로 차별화 권장. **점프/슬라이드 전용 타입(구 LOW/HIGH/PIT) 없음 — 모두 줄 이동으로 회피.**

| 종류 | 현재 임시 | 최종 스프라이트 | 회피 |
|------|----------|-----------------|------|
| BLOCK | 줄을 막는 사각 블록(테마 색) | 벽/문/바위 등 월드 소품 | 안전 줄 이동 |
| MOVER | 줄을 옮겨다니는 블록(슬라럼) | 이동 소품(굴러오는 통 등) | 안전 줄 따라가기 |

## 7. 배경 / 환경 (6월드 테마, 2D 패럴럭스)

> 구조: **원경(하늘/실루엣) + 근경(소품) + 3줄 트랙(바닥·구분선)** 을 수평 스크롤(패럴럭스). 색상 테마는 코드 적용, 최종은 월드별 타일/소품 세트로 교체.

| 레이어 | 현재 임시 | 최종 스펙 | 스크롤 |
|--------|----------|-----------|--------|
| 원경 | 단색 그라디언트 | 월드 배경 타일(이음새 없는 가로 반복) | 느림(0.2~0.4×) |
| 근경 | 단색 띠 | 소품 타일(나무/기둥 등) | 중간(0.6×) |
| 3줄 트랙 | 바닥 밴드 + 줄 구분선 | 바닥 타일 | 1×(월드X) |

- 월드 테마 색(`WorldTheme.bg/bgDark/floor/wallA/wallB` — 구 표기 track/line 정정)은 `data/worlds.ts`. env는 WebP·높이≤512 최적화 완료.
- 스테이지 인트로 배경은 **기존 `world_N_*.webp` 재사용**.

## 11. 에셋 디렉터리 / 네이밍 규약

> 로직은 엔티티 `id` 키만 참조, 실제 파일 경로는 스프라이트 매니페스트에서 해석. 교체 시 로직 무변경.

```
public/assets/
  icons/    <name>.webp        // ✅ 기존 22종 (UI 아이콘·테마)
  images/   <name>.webp        // ✅ 기존 13종 (스테이지·보상·타이틀)
  sprites/  <group>.png|webp   // ◻ 신규: 엔티티 스프라이트 아틀라스
            <group>.json       // ◻ 프레임 맵(아틀라스 좌표)
  audio/    <soundId>.ogg/.mp3 // ◻ 폴리싱 단계
```

- 스프라이트 그룹 예: `player`, `mob_school`, `boss_ghost_teacher`, `fx_common`, `obstacles`, `env_school`.
- 네이밍: 소문자 스네이크. 월드 접미사 `_school/_zombie/_lab/_sea/_dracula/_skull`.
- **폐기**: `public/assets/models/`(3D GLB) — 디렉터리·용도 제거. GLB/glTF 파일은 커밋하지 않는다.
