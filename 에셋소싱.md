# 🎨 몬스터 헌터 러너 — 에셋 소싱 명세서

> **목적**: 전체 6월드에 필요한 모든 에셋의 **목록·형식 스펙·교체 방침**을 정리한다.
> **원칙**: 현재 빌드는 **전부 프리미티브 도형/CSS/이모지**로 동작하며(에셋 0개), 시각·로직(view-logic)이 분리되어 있어 GLTF 모델·스프라이트·오디오를 **교체만으로** 반영한다.
> **렌더링 컨셉**: 2.5D — 3D 배경 + 프리미티브/스프라이트, 서브웨이 서퍼 스타일 3인칭 뒤따라가기 카메라.
> **버전**: 2.0 · **기준일**: 2026-06-11
> **v2.0 변경**: 1월드 MVP → **6월드 전체 구현** 반영. 몬스터 18종(월드별 3종, 도형+빨간 눈 차별화), 보스 12종(범용 패턴 엔진 + 보스별 시그니처/색/탄 모양), 월드별 환경·장애물 테마, 보상 장비 6종(망토 2/모자 4), 사운드 ID를 실제 코드(`systems/Sound.ts`)와 동기화.

## 0. 공통 형식 / 기술 스펙

### 0.1 디바이스 / 성능 예산
- 타깃: 중급폰 이상(Galaxy S / iPhone 12+ 급), 세로(Portrait) 모바일 브라우저.
- 목표 프레임레이트: 60fps(최소 30fps 하한).
- 동시 화면 폴리곤 예산(권장): 캐릭터+몬스터+보스+장애물 합산 **~150K tri** 이하.

### 0.2 3D 모델
| 항목 | 스펙 |
|------|------|
| 포맷 | **glTF 2.0 (.glb 권장, 바이너리 단일 파일)** |
| 좌표 | +Z 전진, Y-up, 1 유닛 = 게임 1.0 유닛(레인 간격 2.0 기준). ⚠️ 레인 X 매핑은 `laneX()` 단일 함수 — 카메라가 +Z를 바라보므로 레인0(좌)=+X |
| 폴리곤 | 캐릭터/보스 8~20K tri, 몬스터 1~3K tri, 장애물/아이템 0.2~1K tri |
| 텍스처 | PBR(Albedo/Normal/Roughness), 2048² 이하(모바일은 1024² 권장), PNG/KTX2 |
| 리깅 | 캐릭터·보스는 스켈레톤 + 스키닝. 망토·모자는 본/어태치먼트로 교체 가능하게 |
| 피벗 | 발밑(바닥 접점) 기준, 정면 −Z(플레이어를 바라보는 방향) |

### 0.3 2D 스프라이트 (2.5D 요소·이펙트·아이콘)
| 항목 | 스펙 |
|------|------|
| 포맷 | PNG(알파) 또는 스프라이트 시트(아틀라스 + JSON) |
| 해상도 | @2x 기준 제작(레티나), POT 권장 |
| 빌보드 | 카메라 향하는 빌보드 평면에 매핑(스프라이트형 몬스터·이펙트) |

### 0.4 사운드 (★ 인터페이스만 구현, 에셋 미포함)
| 항목 | 스펙 |
|------|------|
| 포맷 | **.ogg(주) + .mp3(폴백)**, 44.1kHz |
| BGM | 루프 가능, 모노/스테레오, -14 LUFS 내외 |
| SFX | 짧은 단발, 클리핑 없음, 정규화 |
| 구현 | `systems/Sound.ts` `SoundManager`(`play/stop/setVolume/setMuted`)만 구현, 무음 동작. 호출 지점은 전 코드에 배선 완료 — 파일만 채우면 됨 |

### 0.5 라이선스 정책
- 상용/배포 가능 라이선스만 사용(CC0 / CC-BY / 직접 제작 / 구매 에셋).
- 출처·라이선스·작가를 §12 부록(또는 `에셋출처.csv`)에 기록. CC-BY는 크레딧 표기 의무.
- 어린이 대상 콘텐츠 — 과도한 공포·폭력·선혈 표현 지양(전 월드 '귀엽게 으스스한' 톤 유지. 좀비·드라큘라·해골도 카툰풍).

## 2. 일반 몬스터 (월드별 3종 × 6월드 = 18종)

> 현재 임시: 종류별 **서로 다른 프리미티브 도형 + 빨간 눈 2개(적대 신호) + 위협 흔들림 모션**.
> 도형/색은 `data/worlds.ts` `MonsterDef`(shape/color)에서 정의 — 교체 시 이 정의와 매니페스트만 수정.
> 공통 디자인 규칙: ① 수집 아이템과 한눈에 구분(눈·움직임), ② 행동 타입별 실루엣 통일([0] 덩치 탱커 / [1] 날렵한 직진형 / [2] 작은 위빙형).

| 월드 | [0] 느린 탱커 | [1] 직진형 | [2] 위빙형(최약, 튜토리얼용) |
|------|---------------|------------|------------------------------|
| 🏫 학교 | 책 유령 — 상자 `0x8b5a2b` → 떠다니는 책+유령 꼬리 | 연필 유령 — 원뿔 `0xffd54a` → 연필 유령 | 종이비행기 귀신 — 사면체 `0xffffff` → 종이비행기+유령 얼굴 |
| 🧟 좀비 | 좀비 — 캡슐 `0x4d7c0f` → 카툰 좀비 | 좀비 개 — 상자 `0x365314` → 네발 좀비 개 | 까마귀 — 사면체 `0x1c1917` → 까마귀 |
| ⚡ 연구소 | 전선 골렘 — 상자 `0x607d8b` → 케이블 골렘 | 스파크 봇 — 가시다면체 `0x22d3ee` → 전기 로봇 | 드론 — 사면체 `0xa78bfa` → 소형 드론 |
| 🌊 심해 | 복어 — 가시다면체 `0xfb923c` → 가시 복어 | 상어 지느러미 — 사면체 `0x64748b` → 상어 | 해파리 — 원뿔 `0xf9a8d4` → 해파리 |
| 🦇 드라큘라 | 구울 — 캡슐 `0x78716c` → 구울 | 늑대 — 상자 `0x57534e` → 늑대 | 박쥐 — 사면체 `0x292524` → 박쥐 |
| 💀 해골왕 | 뼈 골렘 — 상자 `0xd6d3d1` → 뼈 골렘 | 해골 병사 — 캡슐 `0xe7e5e4` → 해골 병사 | 해골 새 — 사면체 `0xa8a29e` → 해골 새 |

- 스탯(HP/접촉 데미지/EXP)은 월드가 올라갈수록 점증 — 모델 스케일도 월드별 위협감을 반영해 약간씩 키우는 것 권장.
- 보스 소환 잡몹은 위 18종 재사용(별도 에셋 불필요).
- 공통: 피격 플래시(흰색), 처치 시 디졸브/파티클(임시: 즉시 제거 + 데미지 숫자).

## 3. 보스 (월드별 중간+최종 × 6월드 = 12종)

> 보스는 **범용 패턴 엔진**(`entities/Boss.ts`)으로 구동되고, 정의는 전부 `data/worlds.ts` `BossDef`.
> 외형은 `visual: BossPartDef[]`(프리미티브 조립 데이터) → glb 교체 시 이 부분만 모델 참조로 대체.
> 패턴별 **색(`color`)·탄 모양(`projShape`)**이 데이터에 있으므로, 이펙트 에셋도 보스별 색 변형으로 제작.

### 3.2 패턴 타입별 공용 이펙트 에셋 (보스별 색 변형)

| 패턴 타입 | 현재 임시 | 최종 스펙 | 회피 학습 신호 |
|-----------|----------|-----------|----------------|
| projectile (투사체) | 색상 구체/막대/파편(`ball/rod/shard`) | 탄 모양 3종 × 색 변형 메시/스프라이트 | 대상 레인 바닥 마커 |
| barrage (연속 탄막) | 위와 동일, 순차 발사 | 동일 + 연사 머즐 이펙트 | 전 레인 마커 → 발사 레인 순차 강조 |
| wave (지면 파동) | 가로 발광 박스 | 지면 충격파 셰이더/스프라이트(색 변형) | 전 레인 바닥 발광 |
| walls (차단 벽) | 색상 박스(등장/소멸 스케일) | 테마 벽 메시(촉수/묘비/박쥐 떼/뼈 등 월드별) | 대상 레인 그림자 |
| chase (추적 강타) | 따라오는 바닥 마커 + 폭발 링 | 추적 데칼 + 강타 버스트 파티클 | 마커 추적 → 락 시 점멸 |
| rush (보스 돌진) | 보스 본체 고속 이동 + 흔들림 | 돌진 트레일/모션 블러 | 돌진 레인 마커 + 보스 이동 |
| summon (소환) | 확대되는 보라 링 | 소환진 데칼/스프라이트 | 소환진 원 확대 |
| teleport (순간이동) | 깜빡임 페이드 | 디졸브/잔상 이펙트 | 사라짐 0.3~0.5s |
| scream (봉인 비명) | 화면 진동 + 가장자리 암전(CSS) | 음파 링 이펙트 + 포스트프로세스 | 예고 중 진동·암전 |

### 3.3 보스 공통 연출 에셋
- 경직(약점): 빨강 윤곽 발광(아웃라인, 현재 BackSide 캡슐) + "약점!" 팝업 + 데미지 숫자 확대·노란색.
- 페이즈 전환: 화면 플래시, 체력바 색 변경(초록→주황→빨강) + 페이즈 눈금. 최종 페이즈 진입 시 배경 암전(월드 테마 `bgDark`).
- 등장: 상공 강하 인트로 + 이름 배너.

## 4. 장애물 (월드 테마 색 적용)

> 4타입 공통 메시 + **월드별 색상 테마**(`WorldTheme.obsLow/obsHigh/obsBlock`, `applyObstacleTheme()`).
> 최종 교체 시에는 색뿐 아니라 월드별 소품 메시로 차별화 권장.

| 타입 | 현재 임시 | 최종 스펙(월드별 소품 예) | 회피 |
|------|----------|---------------------------|------|
| `LOW` 낮은 장애물 | 낮은 박스(테마색) | 학교: 책상 더미 / 좀비: 무너진 울타리 / 심해: 산호 / 해골: 뼈 무더기 | 점프/레인이동 |
| `HIGH` 높은 장애물 | 머리 위 가로 박스(테마색) | 학교: 현수막 / 연구소: 배관 / 드라큘라: 샹들리에 | 슬라이드/레인이동 |
| `PIT` 구덩이 | 바닥 검은 평면 | 함몰 바닥/구덩이 메시(월드 공용 가능) | 점프 |
| `BLOCK` 전체 벽 | 레인 봉쇄 벽(테마색) | 학교: 사물함 / 좀비: 바리케이드 / 해골: 석벽 | 안전 레인 이동 |

- 패턴 P1~P10은 위 4타입 조합(로직). 별도 에셋 불필요, 배치만.

## 8.5 튜토리얼 / 온보딩 전용 에셋

> 6단계 안전 학습(무피해·무EXP·대기-성공) 구현 완료. 몬스터 학습 단계는 **현재 월드의 최약체**를 사용하므로 별도 에셋 불필요.

| 단계 | 학습 | 필요 에셋 | 현재 임시 |
|------|------|-----------|----------|
| 1 | 자동 달리기 | 안내 텍스트 | 중앙 펄스 텍스트 |
| 2 | 레인 이동 | 스와이프 손동작 / `←→` 키캡, 유도 보석 | 텍스트 + 보석 |
| 3 | 점프 | 위 스와이프 / `↑` 키캡, 느린 `LOW` | 텍스트 + 허들 |
| 4 | 슬라이드 | 아래 스와이프 / `↓` 키캡, 느린 `HIGH` | 텍스트 + 가로막 |
| 5 | 자동 사격 | 약한 몬스터 1기(월드 최약체 재사용) | 텍스트 + 몬스터 |
| 6 | 스킬 안내 | `AUTO` 배지 글로우 / `Q`·`E` 펄스 | 텍스트 |

- 공통: **스와이프 손동작 4방향 애니메이션**, **키캡 아이콘 세트**, **"준비 완료!" 연출**, `건너뛰기` 버튼(구현 완료).
- i18n: 모든 안내는 텍스트 키(ko/en) — 이미지에 텍스트 굽지 말 것.

## 9. 사운드 목록 (★ 인터페이스만, 추후 소싱)

> **사운드 ID = `systems/Sound.ts`의 `SoundId` 유니언과 1:1** (코드 동기화 완료, **호출 지점 전부 배선 완료** — `jump`/`slide`/`laneMove`는 `Player`에서, `bgmRun`/`bgmBoss`는 `Game.setBgm()`으로 상태별 배선됨).

| 플레이어 | `jump` / `slide` / `laneMove` / `hitPlayer` / `levelUp` | `sfx_player_*` | |
| 전투 | `shoot` / `hitMonster` / `kill` | `sfx_combat_*` | |
| 스킬 | `skillBlast` / `skillDash` | `sfx_skill_*` | 연사 폭주/회복 파동은 해금 시 추가 |
| 보스 | `bossIntro` / `telegraph` / `scream` / `bossHit` / `bossStagger` / `bossPhase` / `bossDefeat` | `sfx_boss_*` | `telegraph`는 전 패턴 공용 예고음 |

| 흐름 | `checkpoint` / `victory` / `gameover` | `sfx_flow_*` | |

- 보스 12종 차별화가 필요하면 `telegraph`/`bossHit`에 월드 변형(예: `sfx_boss_telegraph_skull`)을 매니페스트에서 매핑 — 코드 ID는 유지.

## 11. 에셋 디렉터리 / 네이밍 규약

> 로직은 엔티티 `id` 키만 참조하고, 실제 파일 경로는 에셋 매니페스트(맵)에서 해석한다. 교체 시 로직 무변경.

```
public/assets/
  models/
    player.glb
    monsters/   # monster_<id>.glb — monster_bookGhost.glb, monster_zombie.glb …
    bosses/     # boss_<id>.glb   — boss_ghostTeacher.glb, boss_skullKing.glb …
    items/      # item_coin.glb, cape_ghost.glb, hat_zombie.glb …
    obstacles/  # obstacle_low_<world>.glb …
  sprites/      # fx_*.png, ui_*.png (아틀라스 + JSON)
  textures/     # track_<world>.png, wall_<world>.png …
  audio/
    bgm/        # bgm_run.ogg, bgm_boss.ogg (+ .mp3 폴백)
    sfx/        # sfx_*.ogg
  fonts/        # 한글+영문 게임 폰트(라이선스 확인)
```

- 네이밍: `<카테고리>_<식별자>[_<변형>].<ext>`, 소문자(식별자는 코드 id 그대로). 예) `monster_paperGhost`, `boss_seaWitch_phase3`, `fx_burst_chase`.
- **코드 식별자 단일 소스 ↔ 에셋 키 매핑**:

| 코드 식별자 | 정의 위치 | 에셋 키 예 |
|-------------|-----------|-----------|
| 몬스터 18종 `MonsterDef.id` (`bookGhost` … `skullBird`) | `data/worlds.ts` `WORLDS[].monsters` | `monster_<id>` |
| 보스 12종 `BossDef.id` (`ghostTeacher` … `skullKing`) | `data/worlds.ts` `WORLDS[].midBoss/finalBoss` | `boss_<id>` |
| 보상 장비 6종 (`ghost_cape` … `skull_crown`) | `data/worlds.ts` `REWARD_ITEMS` | `cape_*` / `hat_*` |
| 장애물 `LOW`/`HIGH`/`PIT`/`BLOCK` | `entities/Obstacle.ts` (+월드 색 `WorldTheme.obs*`) | `obstacle_<type>_<world>` |
| 수집물 `coin`/`gem`/`heal` | `entities/Pickup.ts` | `item_*` |
| 적 탄 모양 `ball`/`rod`/`shard` | `data/worlds.ts` `EnemyProjShape` | `proj_<shape>` (+색은 머티리얼) |
| 패턴 이펙트 9종 | `entities/Boss.ts` 패턴 타입 | `fx_<patternType>` |
| 월드 6종 (`school` … `skull`) | `data/worlds.ts` `WorldDef.id` | 배경·BGM 변형 접미사 `_<world>` |
| 사운드 | `systems/Sound.ts` `SoundId` | §9 표 |
