# 🖌️ 2D 에셋 작업 지시서 (sprite-gen 애니메이션)

> **버전**: 2.1 · **기준일**: 2026-07-13 · **파이프라인/현황**: `에셋고도화_계획.md` · **원 범위**: `에셋소싱.md`
> **도구**: `sprite-gen`(Codex 스킬) — base image 1장 + action list → 아틀라스 + `manifest.json`(+GIF/contact sheet QA).
> **원칙**: 아트=Codex/sprite-gen, 재생=게임코드. 계약은 **§3 manifest 스키마**. 매니페스트가 없으면 게임은 정적 폴백하므로 **드롭인 즉시 애니 활성**.
> **전략(확정)**: 기존 정적 PNG에 얽매이지 않는다 — **전면 재생성 허용**. 엔티티당 sprite-gen 1회 실행으로 **hero 정지컷 + 애니 프레임을 동시 산출**해 화풍·크기 편차를 원천 제거한다(기존 에셋은 교체 전까지 임시 폴백).

## 0. 공통 규칙 (전 에셋)

### 0.1 화풍 락 (가장 중요 — 톤 통일)
- **새 스타일 레퍼런스 1장을 먼저 확정**(P1에서 플레이어로 검증): 기존 `title_bg`/`world_*`/`reward_*` webp 톤 = **귀엽게 으스스한 할로윈 카툰**(페인터리, 도톰한 외곽선, 채도 높음, 어린이 친화, 선혈·과공포 금지).
- 이후 **모든 엔티티를 그 레퍼런스 기준으로 재생성**. 개별 프롬프트로 처음부터 만들지 말 것(톤 드리프트 원인). 기존 에셋 유지 여부는 재생성본과 나란히 비교해 결정.
- **상단 광원** 고정(위 밝고 아래 어둡게), 외곽선 두께 일정.

### 0.2 기술 스펙 (엔진 = Canvas 2D, 논리 960×540, ppu 24)
| 항목 | 규칙 |
|------|------|
| 포맷 | 아틀라스 **PNG(알파)** 기본. WebP(알파)도 가능 — 단 **`manifest.image`에 `.webp` 확장자를 명시**해야 로더가 인식 |
| 시점/방향 | **정측면, 우향**. 게임이 몹·보스는 좌향으로 자동 반전(`flip`) |
| 피벗 | **발밑 = 프레임 하단 중앙**. 모든 프레임에서 발 위치 일정(흔들리지 말 것) |
| 배경/그림자 | **넣지 말 것** — 접지 그림자는 엔진이 그림 |
| 프레임 정합 | 한 state의 모든 프레임은 **동일 캔버스·동일 피벗**(정렬 튐 금지) |

### 0.3 표시 크기 (게임이 height로 고정 — 소스 해상도 무관하나 비율은 맞출 것)
- 플레이어 **88px**. 몹: 탱커 89 / 직진 72 / 위빙 58px. 장애물 BLOCK 72 / MOVER 62px.
- **보스**: 소스 해상도 224~256px 권장이나, **실제 표시 높이는 `BossDef.visual` 바운딩박스 기반 약 75~100px**(보스마다 다름). 즉 소스가 커도 화면에선 축소된다 — 실루엣이 축소 후에도 읽히게.
- 소스는 표시의 **2~3배**로 제작 후 sprite-gen이 정리.

## 1. 애니메이션 액션 리스트 (state·fps·loop·프레임수)

> sprite-gen `action list`에 이대로 넣는다. state 이름은 **게임이 요청하는 키와 정확히 일치**해야 한다(불일치 시 idle 폴백).

### 1.1 플레이어 (`player`) — 최우선 P1
| state | fps | loop | 프레임 | 내용 |
|-------|----:|:----:|:-----:|------|
| `run` | 12 | ✔ | 6 | 달리기 사이클(발·팔·검 흔들림). **기본 표시 상태** |
| `jump` | 12 | ✗ | 3 | 웅크림 1 → 상승 1 → 하강 1 (loop=false, 마지막 고정. 체공 0.55s에 맞춰 fps≈5~6도 가능) |
| `hit` | 12 | ✗ | 2 | 피격 움찔(붉은 플래시 톤) |
| `idle` | 6 | ✔ | 4 | **폴백 전용**(게임은 요청하지 않음 — 요청 state가 매니페스트에 없을 때 대신 재생). 선택 |
| `death` | 10 | ✗ | 3 | 쓰러짐 — **현재 미배선**(제작해도 재생 안 됨, 향후 배선). 선택 |

- **소켓 필수**(§2): 게임이 표시하는 **`run`/`jump`/`hit` 모든 프레임**에 `head`(모자), `back`(망토). (idle 제작 시 idle에도)

### 1.2 일반 몹 (`mob_<world>_<id>`) — P3
| state | fps | loop | 프레임 | 비고 |
|-------|----:|:----:|:-----:|------|
| `walk` | 8~10 | ✔ | 4 | **기본 표시 상태**(게임이 요청하는 유일한 state). 티어별 모션: 탱커=무겁게, 직진=빠르게, 위빙=둥실 |
| `death` | 10 | ✗ | 3 | **현재 미배선**(향후). 선택 |
- 18종 전부보다 **대표 3종(탱커/직진/위빙) 먼저** 검증 후 확장.

### 1.3 보스 (`boss_<id>`) — P4
| state | fps | loop | 프레임 | 비고 |
|-------|----:|:----:|:-----:|------|
| `idle` | 6 | ✔ | 2~3 | **기본**(게임이 요청하는 유일한 state). 호흡/부유 |
| `attack` | 10 | ✗ | 3~4 | **현재 미배선**(향후 예고 타이밍용). 선택 |
| `hit` | 12 | ✗ | 2 | **현재 미배선**(향후). 선택 |
- 아틀라스 파일명은 단일 PNG(`boss_<id>.png`)와 **겹치지 않게** `manifest.image`를 `boss_<id>_atlas.png`로 지정.

### 1.4 FX/발사체 (선택, P4)
- `fx_proj_ball/rod/shard` 회전 4f, `fx_wave`/`fx_scream` 확산 3~4f 등. 없으면 현재 정적 사용.

## 2. 부착물 소켓 (모자/망토 앵커링) — P2

**왜**: 지금은 모자·망토가 96×96 전신 오버레이라 몸 애니를 안 따라가고 정렬이 어긋남. → **소켓 앵커**로 전환.

**산출물**
1. **소형 아이템 스프라이트**(전신 X, 아이템만 + 자체 피벗):
   `attach_hat_<id>.png`(피벗 아래중앙), `attach_cape_<id>.png`(피벗 목/어깨). id는 `ghost/dracula`(cape), `zombie/lightning/seawitch/skull`(hat).
2. **플레이어 매니페스트 `sockets`**: `run`/`jump`/`idle`의 **프레임마다** `head`(정수리), `back`(어깨/등) 좌표(프레임 로컬 px).

**게임 사용**(런타임 이미 구현): `socketScreenPos('player', state, frame, 'head'|'back', {cx,anchorY,height,flip})` → 그 위치에 아이템 blit. 소켓이 없으면 자동으로 현재 전신 오버레이 유지.

## 3. manifest.json 계약 (필수 준수)

```jsonc
{
  "image": "player_atlas.png",        // 아틀라스(같은 폴더). §5 규칙: 항상 <group>_atlas.png 명시. WebP면 .webp 확장자 명시
  "pivot": "bottom",
  "states": {
    "run": { "fps": 12, "loop": true, "frames": [ {"x":0,"y":0,"w":96,"h":96}, {"x":96,"y":0,"w":96,"h":96} ] }
  },
  "sockets": { "run": [ {"head":[48,8],"back":[34,42]}, {"head":[49,7],"back":[35,41]} ] }   // 선택, frames와 길이 일치
}
```
- `frames`=아틀라스 절대 px. state 이름=§1 키. sprite-gen 원본 필드명이 다르면 **변환기로 이 스키마에 맞춰 저장**(§7).
- 파일 위치: `public/assets/sprites/<group>.json` + `<group>_atlas.png`.

## 5. 네이밍 / 전달

**공통 규칙 — 아틀라스는 반드시 `_atlas` 접미사** (정적 hero PNG와 파일명 충돌 방지. 정적 PNG는 매니페스트 로드 전/실패 시 즉시 표시되는 폴백이므로 함께 유지·재생성한다):

```
public/assets/sprites/
  # 그룹 = 매니페스트 파일명(확장자 제외). 게임이 <group>.json을 fetch한다.
  player.json + player_atlas.png        # 그룹 'player'. 정적 폴백은 player_idle.png(코드 참조명 — 이 이름 유지)
  mob_<world>_<id>.json + mob_<world>_<id>_atlas.png   # 정적 폴백 mob_<world>_<id>.png 유지
  boss_<id>.json + boss_<id>_atlas.png                  # 정적 폴백 boss_<id>.png 유지
  attach_hat_<id>.png / attach_cape_<id>.png            # 소형 부착 아트(P2)
  env_<world>_(far|near|floor).webp     # 완료
```
- **모든 매니페스트에 `"image": "<group>_atlas.png"`를 명시**한다(생략 시 `<group>.png`를 아틀라스로 착각 — 정적 폴백 PNG가 통째로 그려지는 사고 방지).
- 소문자 스네이크. 월드 접미사 `school/zombie/lab/sea/dracula/skull`.
- 재생성 시 **hero 정지컷(=정적 폴백 PNG)과 아틀라스를 같은 실행에서 함께 산출**해 톤을 일치시킨다.

## 6. 우선순위 (전면 재생성 전제)
**P1 새 스타일 레퍼런스 확정 + 플레이어(hero+run/jump/hit)** → **P2 소켓+부착 소형아트** → **P3 몹 재생성+walk(대표 3종→18종)** → **P4 보스 재생성+idle → FX** → **P5 잔여(env 재생성 필요분·정리)**.
근거: 움직임(P1)·착용감(P2)이 체감 최대. 재생성이 허용되므로 화풍 통일은 별도 패치 단계가 아니라 **각 단계에서 hero+애니 동시 산출**로 해결. 크기·용량은 코드에서 이미 정리됨.

## 7. QA / 인수 기준
- state별 GIF·contact sheet로 **모션·정렬·발밑 피벗** 확인(프레임 튐 없음).
- 게임 드롭인 후: 해당 엔티티가 애니 재생 & 폴백 정상, `tsc/test/lint/build` 그린.
- 매니페스트 필드명이 sprite-gen 원본과 다르면 §3 스키마로 변환해 저장.
- 화풍: `artifacts/asset-contact-sheets/`에 나란히 놓고 톤 이상치 색출.
