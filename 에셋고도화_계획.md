## 1. 파이프라인 아키텍처 (역할 분리)

```
[이미지 생성 모델] 기준 1장(hero pose)  ─┐
                                          ├─▶ [sprite-gen / Codex] ─▶ 아틀라스(png/webp) + manifest.json (+sockets)
[에셋작업지시서.md] action list·style ────┘                              │
                                                                          ▼
                                            public/assets/sprites/<group>.png + <group>.json
                                                                          │
                                                                          ▼
                                            [게임 런타임 systems/Sprites.ts]  drawAnim() / socketScreenPos()
                                                                          │  (매니페스트 없으면 단일 PNG → 도형 폴백)
                                                                          ▼
                                            Player / Monster / Boss / Environment.draw()
```

- **경계**: 아트·프레임 산출 = sprite-gen(Codex). 재생·좌표·상태 결정 = 게임 코드. 계약은 **manifest.json 스키마**(§4).
- **하위호환**: 매니페스트가 없으면 모든 애니 API가 폴백 → 현재 정적 스프라이트/도형으로 정상 동작. 즉 **아틀라스를 넣는 순간 자동 애니**.

## 4. manifest.json 계약 (게임이 소비하는 스키마 — 반드시 준수)

파일: `public/assets/sprites/<group>.json`(매니페스트) + `<group>_atlas.png`(아틀라스 — **`_atlas` 접미사 필수**, 정적 폴백 PNG와 충돌 방지).
`group`: `player` · `mob_<world>_<id>` · `boss_<id>`. WebP 아틀라스는 `image`에 `.webp` 확장자 명시(로더 지원).

```jsonc
{
  "image": "player_atlas.png",   // 아틀라스 파일명(같은 폴더) — 항상 명시(생략 시 <group>.png = 정적 폴백과 충돌)
  "pivot": "bottom",             // 세로 앵커(기본 bottom=발밑)
  "states": {
    "run":  { "fps": 12, "loop": true,  "frames": [ { "x":0,   "y":0, "w":96, "h":96 }, { "x":96, "y":0, "w":96, "h":96 } ] },
    "jump": { "fps": 12, "loop": false, "frames": [ { "x":0,   "y":96,"w":96, "h":96 } ] },
    "hit":  { "fps": 12, "loop": false, "frames": [ { "x":96,  "y":96,"w":96, "h":96 } ] }
  },
  "sockets": {                   // (선택) 부착물 앵커 — state별·프레임별, 프레임 로컬 px(프레임 좌상단 기준)
    "run": [ { "head":[48,8], "back":[34,42] }, { "head":[49,7], "back":[35,41] } ]
  }
}
```

- **state 이름(게임이 실제 요청)**: 플레이어 `run`/`jump`/`hit`, 몹 `walk`, 보스 `idle`. 요청 state가 매니페스트에 없으면 `idle`→첫 상태로 자동 폴백. (`idle`은 폴백 전용, `death`/`attack` 등은 **미배선** — 제작해도 아직 재생 안 됨)
- **프레임 사각형**은 아틀라스 내 절대 px. **발밑 하단중앙** 정렬 전제(pivot bottom).
- **소켓**은 선택. 있으면 `socketScreenPos()`로 모자·망토를 앵커(§5). frames 길이와 sockets[state] 길이 일치시킬 것.
- sprite-gen 원본 산출(`sprite-sheet-alpha.png` + `manifest.json.frame_layout`)과 필드명이 다르면 **작은 변환기**로 위 스키마에 맞춰 저장(작업지시서 §7).

## 5. 부착물 앵커링 설계 (모자/망토) — 사용자 제안 채택

**이전 문제(P2 전)**: 모자·망토를 96×96 전신 오버레이로 몸과 같은 앵커에 겹쳐 그려 정렬이 아트에 전적으로 의존했고, 애니 시 몸 흔들림을 따라가지 못했다.

**해결(소켓)**:
1. 아이템을 **소형 스프라이트**로 제작(전신 X): `attach_hat_<id>.png`, `attach_cape_<id>.png` + 자체 피벗(모자=아래중앙, 망토=목/어깨).
2. 플레이어 매니페스트에 **프레임별 `head`/`back` 소켓** 포함.
3. 그리기: `socketScreenPos('player', state, frame, 'head', {cx, anchorY, height, flip})` → 그 위치에 모자 blit. 망토는 몸 뒤, `back` 소켓.
4. 프레임마다 소켓이 이동 → 점프·달리기 중에도 정확히 따라감.

**런타임은 구현 완료**(`socketScreenPos`, `animFrameAt`, `Player.ts` 소켓 우선 렌더). 현재 6종 장착물은 carry-forward 소형 아트이며, 로드 실패 시 전신 오버레이/색상 도형 폴백을 유지한다.

**착용 트리거(2026-07-13 개정)**: 착용 획득/장착은 **스테이지 진행에 자동 연동**한다 — 별도 장착 UI 없이 각 월드의 **중간보스를 처치하면 그 월드 테마 장비(cape/hat)를 즉시 착용**(`Game.equipWorldCosmetic`), RUNNING_2·최종보스 구간부터 외형이 바뀐다. 최종 클리어(REWARD)에서 영구 지급·persist, 다음 런은 `restorePersistedItems`로 복원. 즉 "스테이지를 나아갈수록 장비가 늘어나는" 진행형 외형.

## 7. 리스크 / 주의
- **파일 확장자 규칙**: 기본 PNG. `env_*`는 webp. 그 외 webp는 **manifest.image에 확장자 명시** 시 인식(`extOverride` — 검토수정 반영).
- **아틀라스 이름 충돌**: 정적 폴백 PNG(`<group>.png`)가 이미 존재 → 아틀라스는 **전 그룹 공통 `<group>_atlas.png`** + manifest.image 명시. 생략 시 정적 PNG를 아틀라스로 착각해 통째로 그려지는 사고.
- **비루프 상태 타이밍**: jump/hit 같은 비루프 시퀀스는 **상태 진입 시점 기준 시간**이 필요 — Player는 per-state 타이머(`animT`) 반영 완료. 새 엔티티에 비루프 상태 배선 시 동일 패턴 준수.
- **성능**: 프레임 과다·대형 아틀라스 주의. 아틀라스도 필요 시 WebP.
- **화풍**: 전면 재생성 전략 — 새 레퍼런스 확정 전에는 신규 에셋 생성 보류.
