# 그래픽 에셋 코딩 인수인계

## 1. 개요

- 최종 에셋 위치: `public/assets/sprites/`
- 정적 폴백: PNG 74개 · 애니 아틀라스 21개 · 소켓용 장착물 별칭: 6개
- 미리보기: `artifacts/asset-contact-sheets/all-assets.png`
- QA 기록: `artifacts/asset-qa.md`
- 공통 방향: 우향 정측면. 좌향이 필요하면 Canvas에서 `scale(-1, 1)`로 반전한다.
- 캐릭터·보스·장애물·FX는 RGBA 투명 PNG다.
- 환경은 월드에 따라 RGB 또는 RGBA이며 모두 좌우 반복 경계가 일치한다.

Vite 런타임 경로 예시:

```ts
const SPRITE_BASE = `${import.meta.env.BASE_URL}assets/sprites`;
const src = `${SPRITE_BASE}/mob_school_bookGhost.png`;
```

## 7. 환경 레이어

파일 규칙:

```text
env_<worldId>_far.png
env_<worldId>_near.png
env_<worldId>_floor.png
```

| 월드 | far | near | floor |
|---|---:|---:|---:|
| school | 1536x512 | 1536x512 | 1536x512 |
| zombie | 1024x512 | 1024x512 | 1024x256 |
| lab | 960x540 | 960x540 | 960x540 |
| sea | 1536x512 | 1536x512 | 1536x512 |
| dracula | 1942x809 | 1774x887 | 2172x724 |
| skull | 1024x512 | 1024x512 | 1024x512 |

- `far`: 0.3배 패럴럭스 속도
- `near`: 0.6배 패럴럭스 속도
- `floor`: 1.0배 패럴럭스 속도
- 전 레이어는 좌우 첫·마지막 픽셀 열이 동일해 반복 타일링 가능하다.
- 원본 크기가 월드별로 다르므로 고정 픽셀 크기로 가정하지 말고 목표 밴드 높이에 맞춰 비율 스케일한다.
- `near/floor`가 불투명한 월드도 있으므로 세 레이어를 화면 전체에 단순 중첩하지 않는다. 각 이미지를 원경·근경·바닥의 세로 밴드에 잘라 배치하거나, 엔진 레이아웃에 맞는 소스 영역을 `drawImage`로 지정한다.

## 8. 권장 구현 순서

1. 이미지 캐시/프리로더를 추가하고 월드 시작 전에 필요한 파일을 로드한다.
2. `Environment.ts`의 도형 배경을 `far/near/floor` 반복 타일로 교체한다.
3. `Monster.ts`, `Boss.ts`, `Obstacle.ts`, `Player.ts`의 애니 우선 경로와 정적/도형 폴백을 확인한다.
4. `Projectile.ts`와 보스 위험 연출에 FX 이미지를 연결한다.
5. `Cosmetics.ts`의 보상 ID를 플레이어 부착물 파일명에 매핑한다.
6. 로드 실패 시 기존 Canvas 도형을 fallback으로 유지한다.

## 9. 검증 상태

- 정적 PNG 수량: 74/74
- sprite-gen 아틀라스: 플레이어 1 + 몹 18 + 보스 2, 모두 `*_atlas.png`와 런타임 매니페스트 연결
- component-row 추출/atlas/inspect/score: mob 18종 + boss 2종, 총 20종 run 모두 `ok: true`, score 100 (player 포함 공개 아틀라스는 21개)
- PNG 디코딩: 통과
- 투명 배경·모서리 알파: 통과
- 지상 에셋 하단 중앙 피벗: 통과
- 환경 seamless-x: 18/18 통과
- 프로젝트 테스트: 32개 통과
- 프로덕션 빌드: 통과

이 문서는 에셋 의미와 연결 규칙, 현재 sprite-gen 드롭인 범위를 함께 다룬다. 미생성 그룹은 매니페스트 부재 시 기존 정적 폴백으로 계속 동작한다.
