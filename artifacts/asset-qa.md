# 그래픽 에셋 QA

- 대상: 초기 정적 에셋 세트(`public/assets/sprites/*.png`, 애니메이션 추가 전)
- 기준: `에셋작업지시서.md` v1.0
- 생성 방식: built-in `image_gen` + 크로마키 제거 후처리

## 산출물

- 월드 에셋: 6월드 x 10개 = 60개
- 공통 에셋: 플레이어 1개 + 부착물 6개 + FX 7개 = 14개
- 초기 기준 합계: 74개 (이후 `attach_*` 별칭과 `*_atlas.png`가 추가됨)

## 런타임 검증 가설

1. **매니페스트 누락 가능성**
   - 구분 증거: 지시서의 월드별 몬스터 3, 보스 2, 장애물 2, 환경 3과 공통 14개를 실제 파일 목록과 대조한다.
   - 관찰: 초기 정적 PNG 파일 74개가 존재하며 기대 합계 74개와 일치했다.
   - 결론: 기각.

2. **손상 PNG 또는 잘못된 투명 배경 가능성**
   - 구분 증거: Pillow로 모든 PNG를 실제 디코딩·검증하고, 환경 외 파일의 RGBA 모드·네 모서리 알파·비어 있지 않은 알파 bbox를 읽는다.
   - 관찰: 초기 검증 결과 `validated=74 issues=0`. 이후 추가된 atlas/attach 별칭은 `artifacts/sprite-gen/qa-notes.md`에서 별도로 검증한다.
   - 결론: 기각.

3. **작은 표시 크기에서 실루엣 붕괴 또는 월드 톤 불일치 가능성**
   - 구분 증거: 최종 크기로 합성한 월드별 컨택트시트를 직접 열어 전체 7개 섹션을 확인한다.
   - 관찰: 플레이어/FX, 6개 월드의 몬스터·보스·장애물·환경 레이어가 서로 식별 가능하며 cute-spooky 톤과 월드 팔레트를 유지했다.
   - 결론: 기각.

- 드라큘라 월드의 원본 크기 7개를 지정 캔버스(56~256px)로 정규화했다.
- 정면/3/4뷰로 판정된 보스 12개와 일반 몬스터 4개를 우향 90도 정측면으로 재생성 또는 교정했다.
