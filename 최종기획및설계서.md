# 🎮 몬스터 러너 2D — 최종 기획 및 설계서

> **버전**: 3.1 · **기준일**: 2026-07-09
> **개발 범위**: **6개 월드 전체**. 각 월드는 `달리기 → 전투 → 중간보스 → 추가 달리기 → 최종보스 → 보상 → 결과`의 한 사이클로 완주 가능하며, 성장·꾸미기·아이템·월드 해금 시스템이 모두 연결된다.
> **수치 원칙**: 모든 밸런스 수치는 `data/config.ts`(전역)와 `data/worlds.ts`(월드/몬스터/보스별)의 **단일 소스**에서만 관리·조정한다.
> **v3.1 변경 요약 (v3.0 → v3.1)**: 조작을 **순수 3줄 닷지(dodge)**로 단순화. **점프/슬라이드 제거** — 모든 위협은 **위/아래 줄 이동만으로** 회피한다. 장애물은 "줄 블로커"로 통합(항상 안전 줄 ≥1 보장, 동시 최대 2줄 점유), 보스 `wave`/`scream`도 줄 회피형으로 재설계. 화면 **가로(Landscape) 우선** 확정. (사용자 결정 반영 — §26.)
> **v3.0 대전환 요약 (v2.0 → v3.0)**: 게임 **컨셉·세계관·시스템(6월드·몬스터18·보스12·스킬4·성장·꾸미기·i18n)은 그대로 유지**하되 기술 방향을 전면 교체.
> 1. **렌더링**: Three.js(WebGL) **2.5D** → **HTML5 Canvas 2D**. three 의존성 완전 제거.
> 2. **진행 방향**: 화면 안쪽(+Z) 자동 전진(서브웨이 서퍼) → **오른쪽(+X)으로 달리는 사이드스크롤 러너**.
> 3. **레인**: X축 좌/중/우 3레인 → **Y축 위/중간/아래 3줄(row)**. `laneX()` → `laneY()`.
> 4. **에셋**: 프리미티브 도형/GLTF → **2D 프리미티브(캔버스 도형)로 선(先)동작 → 스프라이트 시트로 후(後)교체**. GLTF 파이프라인 폐기. **기존 2D WebP 35개는 그대로 재사용**(§25).

## 0. 제품 한 줄 정의

> "오른쪽으로 달리고, **위아래 줄을 오가며 피하고**, 싸우고, 성장하며 6명의 몬스터 왕을 물리치는 **6~12세 어린이용 모바일 우선 2D 사이드스크롤 러닝 액션 RPG**"

- **타깃 사용자**: 6~12세 어린이 (막 기기를 다루는 연령 포함). **조작은 위/아래 두 방향 + 자동 전부**로 최대한 단순.
- **타깃 디바이스**: 중급폰 이상 (Galaxy S / iPhone 12+ 급), **가로(Landscape) 화면 우선** — 우측 러너 + 3줄 배치에 최적.
- **스택**: **HTML5 Canvas 2D** + TypeScript + Vite + pnpm (브라우저 실행, 설치 불필요, WebGL·three 미사용).
- **현재 상태**: 6개 월드 전체를 **2D 프리미티브 도형 / CSS / 이모지**로 동작. **기존 2D WebP 에셋(UI 아이콘 22 + 스테이지/보상/타이틀 이미지 13 = 35개)은 그대로 재사용**(§25). 인게임 엔티티만 스프라이트 0개 상태이며, view-logic 분리로 이후 **스프라이트 시트·오디오는 교체만으로** 반영(→ `에셋소싱.md`).

## 1. 전체 비전 (제품 로드맵)

플레이어는 몬스터로 가득한 세상을 **오른쪽으로 달리며**, 앞에서 다가오는 장애물·몬스터를 **위/아래 줄을 오가며 피하고**, 자동 사격으로 처치하고, 아이템을 수집하며 성장한다. 6개 월드를 모두 클리어해 6명의 몬스터 왕을 물리치고 **👑 몬스터 마스터** 칭호를 얻는 것이 최종 목표다.

| 월드 | 테마 | 중간보스 (HP) | 최종보스 (HP) | 클리어 보상 | 슬롯 |
|------|------|----------|----------|-------------|------|
| 🏫 1 | 학교 유령 | 유령 선생님 (400) | 학교 괴담 소녀 (700) | 유령 망토 | cape |
| 🧟 2 | 좀비 마을 | 거대 좀비 (500) | 좀비 킹 (870) | 좀비 모자 | hat |
| ⚡ 3 | 프랑켄슈타인 연구소 | 번개 골렘 (620) | 프랑켄슈타인 (1070) | 번개 헬멧 | hat |
| 🌊 4 | 바다마녀의 심해 | 크라켄 (760) | 바다마녀 (1300) | 바다마녀 왕관 | hat |
| 🦇 5 | 드라큘라 성 | 늑대인간 대장 (920) | 드라큘라 (1560) | 드라큘라 망토 | cape |
| 💀 6 | 해골왕의 성 | 해골 기사 (1100) | 해골왕 (1850) | 해골왕 왕관 | hat |

**6개 월드 전부 대상**. 월드 데이터는 `data/worlds.ts` `WORLDS[]`에 정의되며, 신규 월드/몬스터/보스 추가는 이 파일의 데이터 확장만으로 가능하다.

### 핵심 재미 4축
1. **런닝 닷지** — 우측 자동 달리기 + **위/아래 3줄 이동만으로 회피** (타이밍·리듬 액션)
2. **전투** — 자동 사격(우측 전방) + 수동 스킬(최대 4종), 보스 패턴 회피·약점(경직) 공격
3. **수집** — 동전 / 보석 / 회복 / 보상 장비 / 꾸미기(망토·모자)
4. **성장** — 레벨업 → 능력치 자동 강화 + 월드 진행 → 스킬 해금

## 3. 기술 / 아키텍처

### 3.1 렌더링·루프·좌표 (2D)

- **렌더링**: HTML5 Canvas 2D 컨텍스트(`getContext('2d')`), 단일 페이지 풀스크린 캔버스. `devicePixelRatio` 최대 2 클램프. 논리 좌표(960×540)에 `ctx.setTransform`으로 맞춤(레터박스 허용).
- **드로우 순서(레이어)**: 배경(원경/근경 패럴럭스) → 3줄 트랙 바닥 → 그림자 → 뒤 줄(위) 엔티티 → … → 앞 줄(아래) 엔티티 → 이펙트 → HUD(DOM 오버레이). **줄이 아래일수록(화면 하단, 카메라에 가까움) 나중에 그려 앞에 온다**(간이 깊이 정렬).
- **게임 루프**: `requestAnimationFrame` + **고정 타임스텝(1/60s) 누산기** 업데이트(물리/충돌 안정). 렌더는 프레임마다, 로직은 고정 스텝. `frameDt` 상한 클램프 + **`visibilitychange` 시 자동 일시정지**(백그라운드 슬로모션 방지).
- **좌표계 (2D)**:
  - **worldX**: 진행(달리기) 축. 플레이어 `worldX += runSpeed·dt`. **거리 = 플레이어 worldX**. 값이 클수록 화면 오른쪽에서 접근.
  - **레인(줄) = 화면 Y**: 3줄을 세로로 쌓음. `laneY(lane)`가 화면 기준선 Y를 반환. `lane 0 = 위`, `1 = 중간`, `2 = 아래`. 위협·엔티티도 `lane` 정수만 가진다(점프 높이 같은 연속 Y 없음).
  - **줄 이동 보간**: 인접 줄로 옮길 때 `laneMoveTimer`(≈0.12s) 동안 두 줄의 `laneY`를 lerp해 부드럽게 이동. 이동 중에도 판정은 목표 줄 기준(관용).
- **월드→화면 매핑**: `screenX(worldX) = playerAnchorX + (worldX − camera.scrollWorldX)·PPU`, `screenY(lane) = laneY(lane)`(줄 이동 중이면 lerp). `PPU`(pixels-per-unit)로 월드 단위를 픽셀로 환산. **기준점 SoT = camera.scrollWorldX**(follow에서 = player.worldX).
- **레인 Y 매핑**: `laneY(lane) = trackCenterY + (lane − 1)·laneSpacingPx`. (중간 줄이 세로 중앙 밴드, 위 줄은 그 위, 아래 줄은 그 아래.)
- **에셋 추상화**: 모든 시각 표현은 *시각(view: `draw(ctx)`)* 과 *로직(state)* 을 분리해 "프리미티브 도형 → 스프라이트 시트" 교체 가능.

> **v3.0 대비**: 점프(연속 Y hopY)·슬라이드·그림자 기반 "점프/줄이동 시각 분리"가 **불필요**해졌다(회피는 오직 줄 이동). 각 엔티티는 `lane` 정수 하나로 세로 위치를 완전히 결정하므로 렌더/충돌이 단순해진다. (그림자는 접지감 연출용으로 선택적 유지.)

### 3.3 채택 아키텍처 패턴
- 상태머신 기반 게임 흐름
- 엔티티 view-logic 분리 (**프리미티브 draw → 스프라이트 시트** 교체)
- config 단일 소스 밸런싱 (`data/config.ts` 전역 + `data/worlds.ts` 월드별)
- **범용 패턴 큐 기반 보스 AI** — 9종 패턴 타입을 데이터로 조합 (예고 → 발동 → 후딜/경직)
- 패턴 템플릿 기반 장애물 스폰 (안전 줄 ≥1 보장, 동시 최대 2줄 점유)
- 거리 기반 난이도 램프 (진행도별 패턴 풀/간격 조절)
- 데이터 확장형 월드 (월드/몬스터/보스 추가 = `worlds.ts`만 수정)
- 모듈 구조 (core / entities / systems / ui / data)

## 4. 좌표 / 레인(줄) 사양

- **줄 수**: 3 (0=위, 1=중간, 2=아래)
- **줄 간격**: `laneSpacingPx`(화면 Y, 기준 96px). `laneY(lane) = trackCenterY + (lane − 1)·laneSpacingPx`
- **줄 이동**: 인접 줄 1칸씩, 보간 시간 ≈ 0.12s (세로 이동). 한 번에 1칸(연속 입력은 큐잉).
- **시작 줄**: 중간(1)
- **회피 축**: 위/아래 줄 이동이 **유일한 회피 수단**. (점프·슬라이드 없음.)

| 자동 달리기(우측) | — | — | 속도 자동, 시간에 따라 점증. 플레이어는 화면 좌측 앵커 고정 |
| 위로 줄 이동 | ↑ / W | 위 스와이프 또는 위 줄 탭 | 인접 위 줄로 |
| 아래로 줄 이동 | ↓ / S | 아래 스와이프 또는 아래 줄 탭 | 인접 아래 줄로 |
| 수동 스킬 1·2 | Q(폭발) / E(대시) | 우하단 스킬 버튼 | 쿨다운 기반 (§8) |
| 수동 스킬 3·4 | R(연사) / F(회복) | 스킬 버튼(해금 시 노출) | 월드 해금 스킬 (§8) |
| 일시정지 | Esc / P | ⏸ 버튼 | 오버레이 토글 |

> **조작 단순화**: 회피 입력은 위/아래 2방향뿐. 점프/슬라이드 키·제스처 없음. 스와이프는 방향만 판정(길이·홀드 불필요) → 오입력 최소, 어린이 친화.

### 표준 수치 (`CONFIG.run`)
- 시작 달리기 속도 **12** 유닛/s → 최대 **24** 유닛/s, 가속 **+0.5/s**(선형, 상한 고정). 화면상 속도는 `PPU` 환산.
- 줄 이동 보간 **0.12s**(`lanes.moveTime`).
- 충돌: 데미지(§11 데미지 테이블) + 짧은 무적 **0.5s**(`hitInvuln`).

## 6. 장애물 패턴 (러닝 구간)

스포너는 **패턴 템플릿**을 큐에서 뽑아 오른쪽 전방에 배치한다. 모든 템플릿은 **항상 통과 가능한 안전 줄(≥1)** 을 보장하며, **동시에 최대 2줄만 점유**한다(어린이 난이도 가드 — 3줄을 동시에 막지 않는다). 각 템플릿은 한 "비트(beat)" 분량.

### 6.1 위협 요소
| 요소 | 형태 | 회피 |
|------|------|------|
| `BLOCK` 장애물 | 줄을 막는 정적 블로커(월드 테마 색/형태) | 안전 줄로 이동 |
| `MOVER` 이동 블로커 | 줄을 옮겨 다니는 블로커(슬라럼) | 안전 줄 따라가기 |
| 몬스터 | 줄을 따라 접근(§7.2) | 처치(자동사격) 또는 회피 |
| 픽업 | 줄 위 수집물(동전/보석/회복) | 해당 줄로 지나가며 획득 |

> v3.0의 LOW/HIGH/PIT/BLOCK 4타입(점프/슬라이드 전용)은 **줄 블로커로 통합**. 시각/테마 변형은 유지하되 **회피법은 전부 "안전 줄 이동"** 으로 일원화한다.

### 6.2 패턴 템플릿 (P1~P8, 안전 줄 ≥1 · 동시 ≤2줄)
| ID | 구성 | 해법 | 줄 점유 | 티어 |
|----|------|------|-----------|------|
| P1 단일 블록 | 1줄 `BLOCK` | 옆 줄로 | 1/3 | 초반 |
| P2 이중 블록 | 2줄 `BLOCK`(안전 1줄) | 안전 줄로 | 2/3 | 초·중반 |
| P3 슬라럼 | `MOVER` 위→중간→아래 순차 | 안전 줄 따라가기 | 1/3씩 | 중반 |
| P4 지그재그 게이트 | 이중 블록의 안전 줄이 번갈아 열림 | 리듬 줄 이동 | 2/3 | 중반 |
| P5 보상 라인 | 한 줄 동전 라인(위협 없음) | 그 줄 유지(수집) | 0 | 전 구간 |
| P6 몬스터 + 블록 | 1줄 `BLOCK` + 인접 줄 몬스터 | 이동 후 사격 | 2/3 | 중후반 |
| P7 타이밍 게이트 | 2줄 블록, 안전 줄이 짧게 열렸다 닫힘 | 타이밍 줄 이동 | 2/3 | 후반 |
| P8 몬스터 웨이브 | 여러 줄 몬스터 순차 접근(동시 ≤2 위협) | 사격 + 안전 줄 회피 | 가변 | 후반 |

### 6.3 난이도 램프 (`CONFIG.obstacles.ramp`, 진행도별)
| 진행도 | 출현 패턴 | 패턴 간격 |
|--------|-----------|-----------|
| 0~30% (초반) | P1, P2, P5 | 2.5s |
| 30~70% (중반) | P1~P6 | 1.8s |
| 70~100% (후반) | P2, P3, P4, P6, P7, P8 | 1.2s |

### 6.4 배치 규칙 (`CONFIG.obstacles`)
- **동시에 3줄 전부 점유 금지** — 항상 안전 줄 ≥1(`maxBlockedLanes 2`).
- 연속 동일 템플릿 **2회 초과 금지**(단조로움 방지).
- 줄 이동(≈0.12s) 시간 고려, 다음 위협까지 **최소 회복 간격 0.8s**(`minRecovery`).
- **동시 위협 상한 4**(`maxConcurrentThreats`, 블록+몬스터 합).
- **MOVER 동적 안전-줄 규칙**: (1) MOVER 점유는 매 프레임 '현재 순간 줄' 기준으로 `maxBlockedLanes` 예산에 산입. (2) 레인 전이 구간에는 **출발 줄 + 도착 줄 2줄**을 위협으로 계산하고 그 순간 다른 위협을 배치하지 않는다 → 항상 안전 줄 ≥1. (3) MOVER는 마지막 남은 안전 줄로 진입 금지.
- P5(보상 라인)는 난이도 휴식 + 동전 유도용으로 전 구간 주기 삽입.

## 7. 전투 사양

### 7.1 자동 사격 (`CONFIG.player`)
- 사정거리 **18 유닛**(`fireRange`, worldX 기준 우측 전방) 내 **가장 가까운 몬스터/보스** 자동 타겟. 기본은 **같은 줄 우선**, 없으면 사정거리 내 최근접.
- 발사 간격 **0.4s 고정**(`fireInterval`, 초당 2.5발). 성장 비대상. 스킬 '연사 폭주'만 일시 −50%.

- ⚠️ 플레이어 피격 히트박스는 80% 축소되지만, **자동 사격 명중 판정은 정상 크기 유지**(§13.2).

## 8. 수동 스킬 시스템

> **스킬 풀 4종. 기본 2종 장착, 나머지 2종은 월드 진행으로 해금.** 슬롯제로 향후 확장 여지를 둔다. (점프/슬라이드가 없어도 4종 전부 그대로 유효.)

### 8.1 스킬 풀 (`CONFIG.skills`)
| 슬롯 | 스킬 | 효과 | 쿨다운 | 해금 조건 | 입력 |
|------|------|------|--------|-----------|------|
| 1 | **광역 폭발** (blast) | 우측 전방 범위(14) 몬스터·보스에 `공격력 ×4` 즉시 피해 | 8s | 기본 장착 | Q |
| 2 | **무적 대시** (dash) | 1.5s 무적 + 달리기 속도 +50% + 장애물·몬스터 관통 | 12s | 기본 장착 | E |
| 3 | **연사 폭주** (rapidFire) | 5s간 사격 간격 −50%(연사 2배) | 14s | **3월드 클리어** | R |
| 4 | **회복 파동** (healPulse) | 즉시 체력 +40 + 주변 잡몹 약한 넉백 | 20s | **5월드 클리어** | F |

> **무적 대시**는 "막힌 줄을 잠깐 관통"하는 생존기로, 순수 줄 닷지에서 회피 실수 보정 역할을 그대로 수행한다.

### 8.2 슬롯 / 해금 / 자동 발동
- 슬롯은 **해금에 따라 2 → 3 → 4개로 확장**(교체/로드아웃 UI 없음 — 어린이 대상 단순화).
- 해금 판정: `unlockedWorldIdx() >= CONFIG.skills.unlocks[id]` (연사=3, 회복=5). 미해금 스킬은 HUD 버튼 미노출·입력 무시·자동 발동 제외.
- HUD에 각 슬롯 **쿨다운 원형 게이지** 표시. 쿨다운 중 입력 무시(시각적 비활성).
- **자동 발동 토글(기본 ON)** — §13.1 참조. 자동 모드 시 슬롯에 `AUTO` 배지.
- **해금 연출**: 해당 월드 클리어 REWARD 화면에 "✨ 새 스킬 해금! — <스킬명>". 또한 **다음 판 시작 시 인게임 배너**로 "새 스킬 사용 가능! 🔥 연사 폭주 (R)"를 1회 안내(`mhr_skill_announced`로 중복 방지).

### 8.3 자동 발동 조건 (§13.1, `Combat.updateAutoSkills`)
- **광역 폭발**: 쿨 0 && 우측 전방 사정거리(14) 내 몬스터/보스 존재 시.
- **연사 폭주**: 해금 && 쿨 0 && 버프 미적용 && 전방 적 존재 시.
- **회복 파동**: 해금 && 쿨 0 && `체력비 ≤ 0.55`(`autoHealHpFrac`) && 회복량 낭비 없을 때.
- **무적 대시**: 회피 불가 위협(안전 줄로 제때 이동 불가) 감지 시 우선 발동, 없으면 대기(`autoDashIdleDelay` 3s) 후 발동.

### 8.4 설계 의도
- 광역 폭발/연사 폭주 = 잡몹 정리 + 보스 경직창 폭딜 ("공격 강화")
- 무적 대시/회복 파동 = 회피·생존 보정 ("생존"), 어린이 실수 완화

## 9. 보스전 사양 (범용 패턴 엔진)

**진입**: 구간 목표 거리 도달 → 전용 보스 아레나(스크롤 정지, 보스가 오른쪽에서 등장·정면 프레이밍). 보스 체력바 + 이름 표시.
**구조**: 보스는 *패턴 큐*를 순환 실행. 각 패턴 = `예고(telegraph) → 발동(active) → 후딜(recovery)`.
**경직(stagger)**: 일부 패턴 종료 후 무방비. 이 창 동안 **받는 데미지 ×1.5**(★약점, `staggerDamageMult`).

### 9.1 패턴 타입 9종 (`entities/Boss.ts` + `data/worlds.ts`)
모든 보스는 아래 타입을 데이터로 조합. 색(`color`)·탄 모양(`projShape`)은 보스별 커스텀. **모든 회피는 줄 이동**(점프/슬라이드 없음).
> 2D(줄 기준) 재해석: "레인" = 줄(row), "전 줄" = 3줄 전체. 투사체·돌진은 오른쪽에서 왼쪽으로 진행. **위협은 동시 최대 2줄 점유**(항상 안전 줄 ≥1).

| 타입 | 동작 | 회피 | 경직★ |
|------|------|------|-------|
| `projectile` | 특정 줄 투사체 (1~2줄) | 안전 줄로 이동 | — |
| `barrage` | 연속 탄막(줄을 바꿔가며 순차 난사, 항상 안전 줄 1) | 리듬 줄 이동 | — |
| `wave` | 줄을 순차로 덮는 파동(1→2줄, 안전 줄이 이동) ★ | **안전 줄로 이동** | ★ |
| `walls` | 줄 차단 벽(최대 2줄 지속) | 안전 줄 이동 | — |
| `chase` | 플레이어 줄 추적 → 락 → 강타 | 락 직전 줄 이동 | ★ |
| `rush` | 보스 본체가 예고된 줄로 돌진(1~2줄) | 안전 줄 회피 | ★ (긴 경직) |
| `summon` | 잡몹 소환(18종 재사용) | 처치/회피 | — |
| `teleport` | 순간이동(줄/좌우 재배치, 연계용) | (연계 패턴 따름) | — |
| `scream` | active 동안 자동사격 봉인 + 대상 줄 음파 → 경직 ★ | **안전 줄로 이동**(성공 시 D 0) | ★ |

> **`wave` 재설계(구 '점프' 회피)**: 지면 파동이 **줄을 순차로 덮는다**(예: 위→중간→아래로 번지며, 항상 한 줄은 비어 있음). 플레이어는 **안전 줄로 이동**해 회피. 3줄을 동시에 덮지 않는다.
>
> **`scream` 재설계(구 '슬라이드' 회피)**: active 동안 **자동사격 봉인**(디버프, 회피 불가·짧음) + **대상 줄(1~2)에 음파 히트** → 안전 줄로 이동해 데미지 회피. 이후 **경직 ★**. 수동 스킬은 봉인 중에도 사용 가능(카운터플레이).
>
> **`walls` 판정(v2.0 P0 버그 방지)**: **줄 점유(row-occupancy) 기반** — 벽이 선 뒤 플레이어가 **같은 줄**(`player.lane === wall.lane`)이면 **벽당 1회** 피해(`hitDone` 래치). 좌표 거리 비교로 판정하지 않는다.

| 💀 중간 | 해골 기사 | 1100 | 340 | 방패 돌격(rush) |
| 💀 최종 | 해골왕 | 1850 | 660 | 뼈 폭풍 5연발(barrage) |

각 보스는 2~3페이즈. 페이즈 전환 시 예고 시간 단축(`telegraphMult`) / 간격 단축 / 다줄화(단, 안전 줄 ≥1 유지) / 경직 단축(`staggerMult`)으로 가속. 상세 패턴·페이즈 큐는 `data/worlds.ts`가 단일 소스이며, 에셋 색/탄 매핑은 `에셋소싱.md` §3.

### 9.3 1월드 상세 예시

#### 중간보스 — 유령 선생님 (HP 400, 2페이즈)
| 패턴 | 예고 | 발동 | 회피 | D | 후딜/경직 |
|------|------|------|------|---|-----------|
| 분필 던지기 (projectile) | 0.6s | 1줄 흰 분필(막대, 왼쪽으로 속도 20) | 안전 줄로 이동 | 12 | 후딜 0.4s |
| 칠판 충격파 (wave) ★ | 0.8s | 줄을 순차로 덮는 충격파(안전 줄 이동) | **안전 줄로 이동** | 18 | **경직 1.2s** |
| 유령 소환 (summon) | 1.0s | 잡몹 1~2기 | 처치/회피 | 접촉 8 | 후딜 0.6s |

- **P1(100%→50%)**: 큐 `분필 → 소환 → 충격파`, 간격 1.0s. 충격파 후 1.2s 경직이 주 딜 타이밍.
- **P2(50%→0%, 가속)**: 예고 −25%, 간격 0.7s, 분필 2줄(안전 1), 경직 ×0.75.

#### 최종보스 — 학교 괴담 소녀 (HP 700, 3페이즈)
| 패턴 | 예고 | 발동 | 회피 | D | 후딜/경직 |
|------|------|------|------|---|-----------|
| 순간이동 (teleport) | 사라짐 0.5s → 재등장 0.4s | 위/중간/아래 줄 재배치 후 연계 | (연계 따름) | — | — |
| 머리카락 벽 (walls) | 0.7s | 1~2줄 벽 2.0s 봉쇄(안전 1줄) | 안전 줄 이동 | 접촉 20(줄점유·벽당1회) | 후딜 0.5s |
| 공포의 비명 (scream) ★ | 1.0s | 자동사격 봉인 + 대상 줄 음파 | **안전 줄로 이동**(성공 시 D 0) | 15 | active 후 **경직 1.5s** |

- **P1(100%→66%)**: `머리카락 → 순간이동 → 머리카락`, 간격 1.2s.
- **P2(66%→33%)**: `순간이동 → 머리카락(2줄) → 비명 → 머리카락`, 간격 1.0s. 비명 경직 1.5s가 주 딜.
- **P3(33%→0%, 광폭화)**: 예고 −30%, 간격 0.6s, 콤보 `순간이동+머리카락 → 비명`, 경직 ×0.67.

### 9.4 보스 공통 규칙 / 피드백 디자인
- 예고 동안 회피 입력 가능(줄 이동).
- 경직(★) 창에 사격·스킬을 몰아넣는 것이 핵심 딜링.
- 페이즈 전환 시 짧은 무적 0.5s(`phaseTransition.invuln`) + 패턴 큐 리셋.
- 보스 접촉/투사체 피격 시 플레이어 무적 0.5s.
- **경직 연출**: 보스 외곽 빨강 윤곽 발광 + 머리 위 "약점!" 팝업 + 받는 데미지 숫자 확대·노란색.
- **예고 신호 표준(2D)**: 대상 줄 바닥 마커(투사체) / 덮일 줄 순차 발광(파동) / 대상 줄 그림자(벽) / 추적 마커(chase) / 소환진 확대(summon) / 화면 진동·암전(scream).
- **체력바**: 흰색 잔상(딜레이 감소) + 페이즈 경계 눈금 + 페이즈 색(초록→주황→빨강). 최종 페이즈 진입 시 배경 `bgDark` 톤 암전.

## 10. 성장 시스템

```
몬스터 처치 / 보석 수집 → 경험치 → 레벨업 → 능력치 자동 분배
월드 클리어 → 다음 월드 해금 (+ 3·5월드 클리어 시 스킬 해금)
```

### 10.1 능력치 (`CONFIG.player`)
| 능력치 | 초기값 | 레벨업 효과 |
|--------|--------|-------------|
| 체력 | 100 | +10/레벨 |
| 공격력 | 10 | +2/레벨 |
| 이동속도 | (달리기 속도 영향) | +2%/레벨 |
| 치명타 확률 | 5% | +1%/레벨 (배수 ×2) |

- 경험치 곡선: **필요 EXP = `50 * level`** (`expCurve`, 선형 누적).
- 레벨업은 **자동 분배**. (점프력 능력치는 제거 — 점프 없음.)

### 10.2 DPS / 성장 곡선 검산
- 유효 DPS = `공격력 × 2.5 × (1 + 치명타확률)`, 최대 체력 = `100 + 10*(level−1)`

| 레벨 | 공격력 | 치명타 | 유효 DPS | 최대 체력 |
|------|--------|--------|----------|-----------|
| 1 | 10 | 5% | 26.3 | 100 |
| 3 | 14 | 7% | 37.5 | 120 |
| 5 | 18 | 9% | 49.1 | 140 |
| 8 | 24 | 12% | 67.2 | 170 |

### 10.3 EXP 경제 & 도달 레벨
- EXP 획득원: 몬스터 처치(월드별 5~23, §7.2) / 보석 수집 +10 / 중간보스 120~340 / 최종보스 250~660.
- 몬스터·보스 EXP가 월드 진행에 따라 상승하므로 후반 월드도 유사한 성장 곡선을 유지한다.
- ⚠️ **튜닝 레버**: 곡선을 `40*level`로 완만화하면 도달 레벨 상향. 곡선 변경 시 보스 HP는 도달 레벨에 맞춰 재검산 필요.

| 아이템 | 분류 | 효과 |
|--------|------|------|
| 동전 | 재화 | 누적 → **점수 환산 전용**(상점 없음) |
| 보석 | 재화/경험치 | 수집 시 EXP +10 + 점수 |
| 회복 아이템 | 소비 | 체력 +30 |
| 강화 아이템(일시 버프) | 소비 | **보류** (슬롯·에셋만 예약) |

- **점수 공식**(`CONFIG.score`): `거리×1 + 동전×5 + 보석×20 + 처치×30 + 보스×500`. 최고점수는 **월드별로 기록**.

- **생존 설계**: L1 체력 100 기준 무피해 의도 플레이. 실수 누적 시 위험(체력제 게임오버 → 보스 직전 체크포인트 부활).

## 13. 접근성 / 조작 보조

> 채택 범위: **① 자동 스킬 토글 + ② 입력 관용**. 조작이 위/아래 2방향뿐이라 진입 장벽이 매우 낮다.

### 13.1 자동 스킬 발동 (토글, 기본 ON)
- 장착·해금 스킬이 **쿨다운 종료 즉시 자동 발동**(발동 조건 §8.3).
- **OFF 전환** 시 수동 입력(Q/E/R/F·터치)으로 발동.
- 토글 위치: 타이틀 설정 + 일시정지 메뉴, 게임 중 즉시 전환.
- HUD: 자동 모드일 때 슬롯에 **`AUTO` 배지**.
- **자동 회피 보조**: 자동 모드에서 같은 줄에 회피 불가 위협이 임박하고 안전 줄로 제때 못 옮기면 **자동 무적 대시**(쿨 가용 시)로 보조. 대시 쿨 중이면 안전 줄로 자동 1칸 이동을 시도(어린이 방치 플레이 대비).

### 13.2 입력 관용 (Forgiveness, `CONFIG.accessibility`)
| 항목 | 값 | 설명 |
|------|----|------|
| 입력 버퍼 | 0.15s | 조금 미리 눌러도 가능 타이밍에 자동 발동 |
| 줄 이동 관용 | 이동 중 목표 줄 판정 | 보간 중에도 목표 줄 기준으로 안전 판정(끼임 방지) |
| 히트박스 축소 | 시각의 80% | 플레이어 **피격 판정에만** 적용(자동 사격 명중은 정상 크기) |
| 액션 중 입력 큐 | 1개 | 이동 중 다음 입력 1개 큐잉, 종료 즉시 실행 |

## 14. 튜토리얼 / 첫 실행 온보딩

> `RUNNING_1` 직전의 **안전 학습 구간**(무피해·무게임오버·무EXP). 달리기 속도 7(`world.tutorialSpeed`).

### 14.1 진행 방식
- **한 번에 하나씩** — 성공 전까지 다음으로 안 넘어감.
- **대기-성공(wait-for-success)** — 대상이 느리게 접근/정지, 실패해도 데미지 0, 프롬프트 반복.
- **강한 시각 신호** — 중앙 큰 텍스트 + 입력 힌트(PC 키캡 / 모바일 스와이프) + 대상 줄 하이라이트.

### 14.2 4단계 시퀀스 (`CONFIG.tutorial.steps`)
| # | 학습 | 성공 조건 | 힌트 |
|---|------|-----------|------|
| 1 | 자동 달리기 | 캐릭터 저절로 우측 전진 시작 | "달리기는 저절로 돼요!" |
| 2 | 줄 이동(회피) | 위/아래 줄로 이동해 블록 피하기 + 보석 1개 획득 | "↑ ↓ / 위아래로 피해요!" |
| 3 | 자동 사격 | 약한 몬스터(현재 월드 최약체) 처치 | "공격은 저절로!" |
| 4 | 스킬 안내 | 자동 ON: `AUTO` 배지 / OFF: Q·E 1회 | "스킬도 자동이에요" |

- 완료 후 → "준비 완료!" → `RUNNING_1`. (v3.0의 점프·슬라이드 학습 단계 제거.)

### 14.3 스킵 / 반복
- 우상단 **`건너뛰기`** 상시 → 즉시 `RUNNING_1`.
- **세션 플래그(sessionStorage `mhr_tutorial_seen`)** 로 "이미 봄" 기억 → 같은 세션 재시작 시 자동 스킵. 타이틀 `튜토리얼 다시 보기`로 재실행.

## 15. 게임 흐름 / 상태머신

### 15.1 한 사이클 (월드마다)
```
타이틀(월드 선택) → (튜토리얼) → 스테이지 인트로 → 달리기1 →
[중간보스] → 달리기2(후반 난이도) → [최종보스] → 보상(장비+해금) →
결과(레벨·처치·획득·점수·다음 월드/재시작)
```

### 15.2 상태 정의 / 전환 (`GameStateName`)
| 상태 | 진입 동작 | 종료 → 다음 |
|------|-----------|-------------------|
| `TITLE` | 타이틀 UI, 월드 선택, 설정 | 시작 → 첫 진입 `TUTORIAL`, 이미 봤으면 `RUNNING_1` |
| `TUTORIAL`* | 안전 학습(무피해·무EXP) | 완료/건너뛰기 → `RUNNING_1` |
| `RUNNING_1` | 스테이지 인트로(3s, 탭 스킵) → 구간1 스폰·전투 | 목표 도달 → `MIDBOSS` / 체력0 → `GAMEOVER` |
| `MIDBOSS` | 아레나, 스크롤 정지, 보스 체력바 | HP0 → `RUNNING_2` / 체력0 → `GAMEOVER`(부활) |
| `RUNNING_2` | 구간2(후반 난이도), 스크롤 재개 | 목표 도달 → `FINALBOSS` / 체력0 → `GAMEOVER` |
| `FINALBOSS` | 아레나, 3페이즈 보스 | HP0 → `REWARD` / 체력0 → `GAMEOVER`(부활) |
| `REWARD` | 보상 장비 지급 + 외형 변화 + 다음 월드/스킬 해금 연출 (HUD 숨김) | 탭 → `RESULT` |
| `RESULT` | 결과(레벨/처치/획득/점수/최고점수). 다음 월드/재시작/타이틀 | 선택 → `RUNNING_1`(다음 월드) / `TITLE` |
| `GAMEOVER` | 게임오버 UI, 점수. 부활 가능 시 버튼 | 부활 → 체크포인트 / 재시작 / `TITLE` |
| `PAUSE`(오버레이) | 시간 정지, 상태 보존, 자동 스킬 토글 | 계속 / 타이틀 |

\* `TUTORIAL`은 선택적. 세션 플래그 존재 또는 건너뛰기 시 직행.

### 15.3 월드 진행 / 해금
- 월드 N 클리어(REWARD) → `unlockWorld(N+1)`. `unlockedWorldIdx` = 클리어한 최대 월드 인덱스.
- 타이틀 월드 선택: 6버튼 그리드, 미해금은 🔒 잠금. 결과 화면 "다음 월드 ▶"로 직행 가능.
- 3월드 클리어 → 연사 폭주, 5월드 클리어 → 회복 파동 해금(§8).
- 6월드 클리어 → 결과 화면 "👑 몬스터 마스터 달성!".

### 15.4 체크포인트 / 부활
- **사망 시 보스 직전 체크포인트 부활**. 스냅샷: `playerState · inventory · stats · distance · runElapsed · gameState`.

### 15.5 공통 규칙 / 스폰 (`CONFIG.world`)
- `PAUSE`는 모든 플레이 상태에서 토글 가능(상태 보존, 시간 정지). 백그라운드 진입 시 자동 일시정지.
- 전환 시 입력 버퍼 초기화, 페이드/연출 허용.
- 구간 길이: **segment1 650 유닛 / segment2 850 유닛**.
- 스폰: `spawnAhead 45`(worldX 우측) 앞에서 생성, `despawnBehind 14`(좌측) 뒤에서 제거.
- 몬스터 스폰 간격: 초반 4.0s → 후반 2.0s. 보스 아레나 거리 12.
- **동시 위협 상한 4** + **동시 최대 2줄 점유**(항상 안전 줄 ≥1).

## 16. UI / 화면 설계

- **타이틀**: 게임명, **월드별 최고점수**(선택 월드 기준 🏆), 월드 선택 6버튼(잠금 표시), 시작, 튜토리얼 다시 보기, 설정(자동 스킬·언어).
- **인게임 HUD (RUNNING)**: 체력바 / 점수·거리 / 동전·보석 카운트 / 레벨·EXP바 / 스킬 쿨다운(최대 4슬롯, 원형 게이지·AUTO 배지, 미해금 숨김).
- **보스 HUD**: 보스 이름 + 체력바(페이즈 눈금·색·잔상) + 플레이어 체력·스킬 + "약점!" 팝업.
- **스테이지 인트로**: RUNNING_1 진입 시 월드 이미지·이름 3초 표시(탭 스킵). **기존 `world_N_*.webp` 사용.**
- **결과 화면**: 점수·최고점수(신기록 표시), 도달 레벨, 처치 수, 재화, 다음 월드/재시작/타이틀.
- **게임오버**: 도달 거리·레벨·재화·점수, (가능 시)체크포인트 부활, 재시작.
- **보상 화면**: 획득 장비 렌더 + (해당 시)스킬 해금 문구. **진입 시 HUD 숨김.**
- 터치/마우스 공용. HUD는 노치/세이프존 여백 고려. **가로 모드**에서 스킬 버튼은 우하단, 일시정지는 우상단.

export function laneY(lane) { return CONFIG.render.trackCenterY * CONFIG.render.logicalHeight
                                     + (lane - 1) * CONFIG.render.laneSpacingPx; }

       + (worldX - scrollWorldX) * CONFIG.render.ppu;

### 25.2 신규 제작 대상 — 엔티티 스프라이트 (프리미티브 → 스프라이트, view-logic)

## 26. 조작·회피 설계 결정 (v3.1)

> 사용자 결정 반영. 이 절이 §5·§6·§9 회피 사양의 근거다.

- **점프/슬라이드 제거, 순수 3줄 닷지 채택**: 회피 수단을 **위/아래 줄 이동 하나로 통일**. 이유 — (1) 6~12세 조작 단순화, (2) 2D 사이드뷰에서 "수직 줄 이동"과 "수직 점프"의 시각 충돌 원천 제거, (3) 장애물/보스 회피 학습이 "안전 줄 찾기" 하나로 수렴.
- **파급 규칙**: 장애물 4타입 → 줄 블로커 통합(BLOCK/MOVER), 동시 최대 2줄 점유(안전 줄 ≥1), 보스 `wave`(구 점프)·`scream`(구 슬라이드) → 줄 회피형 재설계, 튜토리얼 6→4단계, config에서 점프/슬라이드/중력/코요테 제거, 데이터 모델 `hopY/isJumping/isSliding` 제거.
- **화면 가로(Landscape) 확정**: 우측 러너 + 3줄 세로 배치에 최적.
