## 1. 전체 비전 (제품 로드맵)

플레이어는 몬스터로 가득한 세상을 달리며 장애물을 피하고, 몬스터를 처치하고, 아이템을 수집하며 성장한다. 6개 월드를 모두 클리어해 6명의 몬스터 왕을 물리치고 **👑 몬스터 마스터** 칭호를 얻는 것이 최종 목표다.

| 월드 | 테마 | 중간보스 (HP) | 최종보스 (HP) | 클리어 보상 | 슬롯 |
|------|------|----------|----------|-------------|------|
| 🏫 1 | 학교 유령 | 유령 선생님 (400) | 학교 괴담 소녀 (700) | 유령 망토 | cape |
| 🧟 2 | 좀비 마을 | 거대 좀비 (500) | 좀비 킹 (870) | 좀비 모자 | hat |
| ⚡ 3 | 프랑켄슈타인 연구소 | 번개 골렘 (620) | 프랑켄슈타인 (1070) | 번개 헬멧 | hat |
| 🌊 4 | 바다마녀의 심해 | 크라켄 (760) | 바다마녀 (1300) | 바다마녀 왕관 | hat |
| 🦇 5 | 드라큘라 성 | 늑대인간 대장 (920) | 드라큘라 (1560) | 드라큘라 망토 | cape |
| 💀 6 | 해골왕의 성 | 해골 기사 (1100) | 해골왕 (1850) | 해골왕 왕관 | hat |

**6개 월드 전부 구현 완료**. 월드 데이터는 `data/worlds.ts` `WORLDS[]`에 정의되며, 신규 월드/몬스터/보스 추가는 이 파일의 데이터 확장만으로 가능하다.

### 핵심 재미 4축
1. **런닝 액션** — 자동 전진 + 점프 / 슬라이드 / 좌우 이동 / 타이밍 액션
2. **전투** — 자동 사격 + 수동 스킬(최대 4종), 보스 패턴 회피·약점(경직) 공격
3. **수집** — 동전 / 보석 / 회복 / 보상 장비 / 꾸미기(망토·모자)
4. **성장** — 레벨업 → 능력치 자동 강화 + 월드 진행 → 스킬 해금

## 3. 기술 / 아키텍처

### 3.1 렌더링·루프·좌표

- **렌더링**: Three.js (WebGL), 단일 페이지 풀스크린 캔버스. `pixelRatio` 최대 2 클램프.
- **게임 루프**: `requestAnimationFrame` + **고정 타임스텝(1/60s) 누산기** 업데이트(물리/충돌 안정). 렌더는 프레임마다, 로직은 고정 스텝.
- **좌표계**: 캐릭터는 +Z 방향 자동 전진. 레인은 X축 분할, 점프는 Y축.
- **레인 X 매핑**: 카메라가 뒤(−Z)에서 +Z를 바라보므로 레인0(좌)=+X. `laneX(lane) = (1 − lane) * spacing`.
- **카메라**: 캐릭터 뒤·위 오프셋 `(0, +4, −7)`, 부드러운 추적(lerp). 모드: `title` / `follow` / `boss`.
- **에셋 추상화**: 모든 시각 표현은 *시각(view)* 과 *로직(state)* 을 분리해 "프리미티브 → 모델" 교체 가능.

## 4. 좌표 / 레인 사양

- **레인 수**: 3 (0=좌, 1=중, 2=우)
- **레인 간격**: 2.0 유닛 → X 위치 = `laneX(lane) = (1 − lane) * 2.0`
- **레인 이동**: 인접 레인 1칸씩, 보간 시간 ≈ 0.12s
- **시작 레인**: 중앙(1)

## 5. 러닝 액션 (코어 루프)

| 동작 | 키보드 | 터치 | 설명 |
|------|--------|------|------|
| 자동 전진 | — | — | 속도 자동, 시간에 따라 점증 |
| 좌/우 이동 | ← / → (A / D) | 좌/우 스와이프 또는 레인 터치 | 인접 레인 이동 |
| 점프 | ↑ / W / Space | 위 스와이프 또는 화면 중앙 탭 | 공중 장애물·구덩이 회피 |
| 슬라이드 | ↓ / S | 아래 스와이프 | 낮은 장애물 통과 |
| 수동 스킬 1·2 | Q(폭발) / E(대시) | 우하단 스킬 버튼 | 쿨다운 기반 (§8) |
| 수동 스킬 3·4 | R(연사) / F(회복) | 스킬 버튼(해금 시 노출) | 월드 해금 스킬 (§8) |
| 일시정지 | Esc / P | ⏸ 버튼 | 오버레이 토글 |

### 표준 수치 (`CONFIG.run`)
- 시작 전진 속도 **12** 유닛/s → 최대 **24** 유닛/s, 가속 **+0.5/s**(선형, 상한 고정)
- 점프: 초기 속도 상승 후 중력 **−25**, 공중 체공 ≈ **0.7s**
- 슬라이드: 지속 **0.6s**, 히트박스 높이 절반
- 충돌: 데미지(§11 데미지 테이블) + 짧은 무적 **0.5s**. 구덩이 미점프 통과 시 큰 데미지.

## 6. 장애물 패턴 (러닝 구간)

스포너는 **패턴 템플릿**을 큐에서 뽑아 배치한다. 모든 템플릿은 **항상 통과 가능한 안전 경로**를 보장한다(어린이용 난이도 가드). 각 템플릿은 한 "비트(beat)" 분량. 장애물 4타입 공통 메시에 **월드별 색상 테마**(`WorldTheme.obsLow/obsHigh/obsBlock`)가 적용된다.

### 6.1 장애물 타입
| 타입 | 형태 | 회피 |
|------|------|------|
| `LOW` 낮은 장애물 | 지면 허들 | 점프 또는 레인 이동 |
| `HIGH` 높은 장애물 | 머리 위 가로막 | 슬라이드 또는 레인 이동 |
| `PIT` 구덩이 | 바닥 구멍 | 점프 |
| `BLOCK` 전체 벽 | 레인 일부 봉쇄 | 안전 레인 이동 |

### 6.2 패턴 템플릿 (P1~P10)
| ID | 구성 | 해법 | 레인 점유 | 티어 |
|----|------|------|-----------|------|
| P1 단일허들 | 1레인 `LOW` | 점프 or 옆 레인 | 1/3 | 초반 |
| P2 가로막 | 1레인 `HIGH` | 슬라이드 or 옆 레인 | 1/3 | 초반 |
| P3 구덩이 | 1레인 `PIT` | 점프 | 1/3 | 초반 |
| P4 2레인 차단 | 2레인 `BLOCK`(안전 1레인) | 안전 레인 이동 | 2/3 | 중반 |
| P5 전열 허들 | 3레인 `LOW` | 전체 점프 | 3/3 | 중반 |
| P6 전열 가로막 | 3레인 `HIGH` | 전체 슬라이드 | 3/3 | 중반 |
| P7 슬라럼 | `BLOCK` 좌→중→우 순차(3비트) | 연속 레인 이동 | 1/3씩 | 중후반 |
| P8 점프-슬라이드 콤보 | `LOW` 후 즉시 `HIGH`(같은 레인) | 점프 후 슬라이드 | 1/3 | 후반 |
| P9 보상 아치 | `LOW` 위에 동전 호 | 점프(회피+수집 동시) | 1/3 | 전 구간 |
| P10 몬스터+장애물 | `BLOCK` 1레인 + 인접 레인 몬스터 | 이동 후 사격 | 2/3 | 후반 |

### 6.3 난이도 램프 (`CONFIG.obstacles.ramp`, 진행도별)
| 진행도 | 출현 패턴 | 패턴 간격 |
|--------|-----------|-----------|
| 0~30% (초반) | P1, P2, P3, P9 | 2.5s |
| 30~70% (중반) | P1~P7, P9 | 1.8s |
| 70~100% (후반) | P4~P10 | 1.2s |

### 6.4 배치 규칙 (`CONFIG.obstacles`)
- 연속 동일 템플릿 **2회 초과 금지**(단조로움 방지).
- **물리적 동시해법 불가 조합**은 같은 레인 0.8s 이내 금지. ⚠️ 현 구현은 **패턴 내부**만 명시 검사하며, 패턴 간(cross-pattern) 충돌은 패턴 간격(≥1.2s)으로 보장한다. 간격을 0.8s 미만으로 낮추려면 명시 검사를 추가할 것.
- 점프(≈0.7s)/슬라이드(0.6s) 동작 시간 고려, 다음 장애물까지 **최소 회복 간격 0.8s**(`minRecovery`).
- **동시 위협 상한 4**(`maxConcurrentThreats`, 몬스터 스폰과 합산).
- P9(보상 아치)는 난이도 휴식 + 동전 유도용으로 전 구간 주기 삽입.

## 7. 전투 사양

### 7.1 자동 사격 (`CONFIG.player`)
- 사정거리 **18 유닛**(`fireRange`) 내 **가장 가까운 몬스터/보스** 자동 타겟.
- 발사 간격 **0.4s 고정**(`fireInterval`, 초당 2.5발). 성장 비대상. 스킬 '연사 폭주'만 일시 −50%.
- 발사체 속도 40, 수명 1.2s. 데미지 = 플레이어 공격력 (치명타 시 ×2).
- 처치 시 경험치 + 동전(1~3) / 보석(20%) 드랍.

## 8. 수동 스킬 시스템

> **스킬 풀 4종. 기본 2종 장착, 나머지 2종은 월드 진행으로 해금.** 슬롯제로 향후 확장 여지를 둔다.

### 8.1 스킬 풀 (`CONFIG.skills`)
| 슬롯 | 스킬 | 효과 | 쿨다운 | 해금 조건 | 입력 |
|------|------|------|--------|-----------|------|
| 1 | **광역 폭발** (blast) | 전방 범위(14) 몬스터·보스에 `공격력 ×4` 즉시 피해 | 8s | 기본 장착 | Q |
| 2 | **무적 대시** (dash) | 1.5s 무적 + 전진속도 +50% + 장애물·몬스터 관통 | 12s | 기본 장착 | E |
| 3 | **연사 폭주** (rapidFire) | 5s간 사격 간격 −50%(연사 2배) | 14s | **3월드 클리어** | R |
| 4 | **회복 파동** (healPulse) | 즉시 체력 +40 + 주변 잡몹 약한 넉백 | 20s | **5월드 클리어** | F |

### 8.2 슬롯 / 해금 / 자동 발동
- 슬롯은 **해금에 따라 2 → 3 → 4개로 확장**(교체/로드아웃 UI 없음 — 어린이 대상 단순화).
- 해금 판정: `unlockedWorldIdx() >= CONFIG.skills.unlocks[id]` (연사=3, 회복=5). 미해금 스킬은 HUD 버튼 미노출·입력 무시·자동 발동 제외.
- HUD에 각 슬롯 **쿨다운 원형 게이지** 표시. 쿨다운 중 입력 무시(시각적 비활성).
- **자동 발동 토글(기본 ON)** — §13.1 참조. 자동 모드 시 슬롯에 `AUTO` 배지.
- **해금 연출**: 해당 월드 클리어 REWARD 화면에 "✨ 새 스킬 해금! — <스킬명>" 노출. 또한 **다음 판 시작 시 인게임 배너**로 "새 스킬 사용 가능! 🔥 연사 폭주 (R)"를 1회 안내(발견성 강화, `mhr_skill_announced`로 중복 방지).

### 8.3 자동 발동 조건 (§13.1, `Combat.updateAutoSkills`)
- **광역 폭발**: 쿨 0 && 전방 사정거리(14) 내 몬스터/보스 존재 시.
- **연사 폭주**: 해금 && 쿨 0 && 버프 미적용 && 전방 적 존재 시.
- **회복 파동**: 해금 && 쿨 0 && `체력비 ≤ 0.55`(`autoHealHpFrac`) && 회복량 낭비 없을 때.
- **무적 대시**: 회피 불가 위협 감지 시 우선 발동, 없으면 대기(`autoDashIdleDelay` 3s) 후 발동.

### 8.4 설계 의도
- 광역 폭발/연사 폭주 = 잡몹 정리 + 보스 경직창 폭딜 ("공격 강화")
- 무적 대시/회복 파동 = 회피·생존 보정 ("생존"), 어린이 실수 완화

## 9. 보스전 사양 (범용 패턴 엔진)

**진입**: 구간 목표 거리 도달 → 전용 보스 아레나(전진 정지, 카메라 보스 정면). 보스 체력바 + 이름 표시.
**구조**: 보스는 *패턴 큐*를 순환 실행. 각 패턴 = `예고(telegraph) → 발동(active) → 후딜(recovery)`.
**경직(stagger)**: 일부 패턴 종료 후 무방비. 이 창 동안 **받는 데미지 ×1.5**(★약점, `staggerDamageMult`).

### 9.1 패턴 타입 9종 (`entities/Boss.ts` + `data/worlds.ts`)
모든 보스는 아래 타입을 데이터로 조합. 색(`color`)·탄 모양(`projShape`)은 보스별 커스텀.

| 타입 | 동작 | 회피 | 경직★ |
|------|------|------|-------|
| `projectile` | 레인 투사체 (1~2레인) | 해당 레인 벗어나기 | — |
| `barrage` | 연속 탄막(순차 다발 난사) | 리듬 회피 | — |
| `wave` | 전 레인 지면 파동 | **점프** | ★ |
| `walls` | 레인 차단 벽(지속) | 안전 레인 이동 | — |
| `chase` | 플레이어 레인 추적 → 락 → 강타 | 락 직전 레인 이동 | ★ |
| `rush` | 보스 본체 돌진 | 레인 회피 | ★ (긴 경직) |
| `summon` | 잡몹 소환(18종 재사용) | 처치/회피 | — |
| `teleport` | 순간이동(연계용) | (연계 패턴 따름) | — |
| `scream` | active 동안 자동사격 봉인 → 경직 | **슬라이드**(성공 시 D 0) | ★ |

> **비명(scream) 타이밍 규칙**: `예고 → active(fireLock 동안만 자동사격 봉인) → 경직(★, 사격 복구·×1.5)`. 봉인과 경직은 **순차**이며 절대 겹치지 않는다. 수동 스킬은 봉인 중에도 사용 가능 → 카운터플레이 제공.
>
> **회피 판정(설계 의도)**: 어린이 관용을 위해 `walls`·`scream`은 **1회 판정/래치** 방식이다. `walls`는 벽이 솟은 뒤 같은 레인 점유 시 **벽당 1회** 피해(BLOCK 장애물과 동일 — 무한 드레인 방지, `hitDone`). `scream`은 active 구간 중 **1프레임이라도 슬라이드**하면 회피 성공(래치).

### 9.3 1월드 상세 예시

#### 중간보스 — 유령 선생님 (HP 400, 2페이즈)
| 패턴 | 예고 | 발동 | 회피 | D | 후딜/경직 |
|------|------|------|------|---|-----------|
| 분필 던지기 (projectile) | 0.6s | 1레인 흰 분필(막대, 속도 20) | 레인 벗어나기 | 12 | 후딜 0.4s |
| 칠판 충격파 (wave) ★ | 0.8s | 지면 충격파 전 레인 | **점프** | 18 | **경직 1.2s** |
| 유령 소환 (summon) | 1.0s | 잡몹 1~2기 | 처치/회피 | 접촉 8 | 후딜 0.6s |

- **P1(100%→50%)**: 큐 `분필 → 소환 → 충격파`, 간격 1.0s. 충격파 후 1.2s 경직이 주 딜 타이밍.
- **P2(50%→0%, 가속)**: 예고 −25%, 간격 0.7s, 분필 2레인(안전 1), 경직 ×0.75.

#### 최종보스 — 학교 괴담 소녀 (HP 700, 3페이즈)
| 패턴 | 예고 | 발동 | 회피 | D | 후딜/경직 |
|------|------|------|------|---|-----------|
| 순간이동 (teleport) | 사라짐 0.5s → 재등장 0.4s | 좌/중/우 이동 후 연계 | (연계 따름) | — | — |
| 머리카락 벽 (walls) | 0.7s | 1~2레인 벽 2.0s 봉쇄 | 안전 레인 이동 | 접촉 20 | 후딜 0.5s |
| 공포의 비명 (scream) ★ | 1.0s | 전 레인 음파(봉인) | **슬라이드**(성공 시 D 0) | 15 | active 후 **경직 1.5s** |

- **P1(100%→66%)**: `머리카락 → 순간이동 → 머리카락`, 간격 1.2s.
- **P2(66%→33%)**: `순간이동 → 머리카락(2레인) → 비명 → 머리카락`, 간격 1.0s. 비명 경직 1.5s가 주 딜.
- **P3(33%→0%, 광폭화)**: 예고 −30%, 간격 0.6s, 콤보 `순간이동+머리카락 → 비명`, 경직 ×0.67.

### 9.4 보스 공통 규칙 / 피드백 디자인
- 예고 동안 회피 입력 가능(레인 이동/점프/슬라이드).
- 경직(★) 창에 사격·스킬을 몰아넣는 것이 핵심 딜링.
- 페이즈 전환 시 짧은 무적 0.5s(`phaseTransition.invuln`) + 패턴 큐 리셋.
- 보스 접촉/투사체 피격 시 플레이어 무적 0.5s.
- **경직 연출**: 보스 외곽 빨강 윤곽 발광 + 머리 위 "약점!" 팝업 + 받는 데미지 숫자 확대·노란색.
- **예고 신호 표준**: 대상 레인 바닥 마커(투사체) / 전 레인 바닥 발광(파동) / 대상 레인 그림자(벽) / 추적 마커(chase) / 소환진 확대(summon) / 화면 진동·암전(scream).
- **체력바**: 흰색 잔상(딜레이 감소) + 페이즈 경계 눈금 + 페이즈 색(초록→주황→빨강). 최종 페이즈 진입 시 배경 `bgDark` 톤 암전.

## 10. 성장 시스템

```
몬스터 처치 / 보석 수집 → 경험치 → 레벨업 → 능력치 자동 분배
월드 클리어 → 다음 월드 해금 (+ 3·5월드 클리어 시 스킬 해금)
```

### 10.1 능력치 (`CONFIG.player`)
| 능력치 | 초기값 | 레벨업 효과 |
|--------|--------|-------------|
| 체력 | 100 | +10/레벨 |
| 공격력 | 10 | +2/레벨 |
| 이동속도 | (전진속도 영향) | +2%/레벨 |
| 점프력 | 기본 | +3%/레벨 |
| 치명타 확률 | 5% | +1%/레벨 (배수 ×2) |

- 경험치 곡선: **필요 EXP = `50 * level`** (`expCurve`, 선형 누적).
- 레벨업은 **자동 분배**.

### 10.2 DPS / 성장 곡선 검산
- 유효 DPS = `공격력 × 2.5 × (1 + 치명타확률)`, 최대 체력 = `100 + 10*(level−1)`

| 레벨 | 공격력 | 치명타 | 유효 DPS | 최대 체력 |
|------|--------|--------|----------|-----------|
| 1 | 10 | 5% | 26.3 | 100 |
| 3 | 14 | 7% | 37.5 | 120 |
| 5 | 18 | 9% | 49.1 | 140 |
| 8 | 24 | 12% | 67.2 | 170 |

### 10.3 EXP 경제 & 도달 레벨
- EXP 획득원: 몬스터 처치(월드별 5~23, §7.2) / 보석 수집 +10 / 중간보스 120~340 / 최종보스 250~660.
- 몬스터·보스 EXP가 월드 진행에 따라 상승하므로 후반 월드도 유사한 성장 곡선을 유지한다.
- ⚠️ **튜닝 레버**: 곡선을 `40*level`로 완만화하면 도달 레벨 상향. 곡선 변경 시 보스 HP는 도달 레벨에 맞춰 재검산 필요(HP는 현실 도달 레벨에서 역산됨).

## 11. 아이템 / 수집 & 경제

| 아이템 | 분류 | 효과 |
|--------|------|------|
| 동전 | 재화 | 누적 → **점수 환산 전용**(상점 없음) |
| 보석 | 재화/경험치 | 수집 시 EXP +10 + 점수 |
| 회복 아이템 | 소비 | 체력 +30 |
| 강화 아이템(일시 버프) | 소비 | **보류** (슬롯·에셋만 예약) |
| 보상 장비 6종 | 장비/보상 | 월드 클리어 확정 보상 (§12) |

- 수집물은 레인 위 배치, 캐릭터 통과 시 획득.
- **드랍**(`CONFIG.pickups`): 동전 처치당 1~3개 / 보석 처치 시 20% + 구간 배치(패턴 15%·라인 30%) / 회복 구간당 1~2개.
- **점수 공식**(`CONFIG.score`): `거리×1 + 동전×5 + 보석×20 + 처치×30 + 보스×500`. 최고점수는 **월드별로 기록**.

### 11.1 데미지 종합 테이블
| 출처 | D | 회피법 | 무적 |
|------|---|--------|------|
| 낮은/높은 장애물 충돌 | 15 | 점프·슬라이드/레인이동 | 0.5s |
| 구덩이 추락 | 30 | 점프 | 0.5s |
| 전체 벽(BLOCK) 충돌 | 15 | 안전 레인 이동 | 0.5s |
| 몬스터 접촉 | 8~20 (월드별) | 처치/회피 | 0.5s |
| 보스 투사체/파동/벽/비명 | 12~30 (패턴별) | 패턴별 회피 (§9) | 0.5s |

- **생존 설계**: L1 체력 100 기준 무피해 의도 플레이. 실수 누적 시 위험(체력제 게임오버 → 보스 직전 체크포인트 부활).

## 12. 꾸미기

- 슬롯: **망토(cape) / 모자(hat)** 둘 다 활성.
- 월드 클리어 보상 6종(`data/worlds.ts` `REWARD_ITEMS`): 유령 망토(cape), 좀비 모자·번개 헬멧·바다마녀 왕관·해골왕 왕관(hat), 드라큘라 망토(cape).
- 장착 → **외형 변화**(임시: 색/부착 콘 / 모델 단계: 본·어태치먼트 메시). 슬롯당 최근 획득 1개 표시.
- 획득 장비는 LocalStorage(`mhr_items`)에 영속 → **새 판에서도 외형 유지**.
- 장비 스탯 효과는 **외형만** 반영(스탯 보류).

## 13. 접근성 / 조작 보조

> 채택 범위: **① 자동 스킬 토글 + ② 입력 관용**.

### 13.1 자동 스킬 발동 (토글, 기본 ON)
- 장착·해금 스킬이 **쿨다운 종료 즉시 자동 발동**(발동 조건 §8.3).
- **OFF 전환** 시 수동 입력(Q/E/R/F·터치)으로 발동.
- 토글 위치: 타이틀 설정 + 일시정지 메뉴, 게임 중 즉시 전환.
- HUD: 자동 모드일 때 슬롯에 **`AUTO` 배지**.
- **PIT 자동 점프 보조**: 자동 모드에서 전방 같은 레인에 `PIT`(구덩이, 데미지 30·점프만 회피)가 임박하면 자동 점프로 넘긴다. 대시 쿨과 무관한 무료 안전망(어린이 방치 플레이 대비). 다른 장애물은 자동 대시(쿨 가용 시)로만 보조.

### 13.2 입력 관용 (Forgiveness, `CONFIG.accessibility`)
| 항목 | 값 | 설명 |
|------|----|------|
| 입력 버퍼 | 0.15s | 조금 미리 눌러도 가능 타이밍에 자동 발동 |
| 코요테 타임 | 0.12s | 착지 직전/직후 입력도 점프로 인정 |
| 히트박스 축소 | 시각의 80% | 플레이어 **피격 판정에만** 적용(자동 사격 명중은 정상 크기) |
| 액션 중 입력 큐 | 1개 | 동작 중 다음 입력 1개 큐잉, 종료 즉시 실행 |

## 14. 튜토리얼 / 첫 실행 온보딩

> `RUNNING_1` 직전의 **안전 학습 구간**(무피해·무게임오버·무EXP). 전진 속도 7(`world.tutorialSpeed`).

### 14.1 진행 방식
- **한 번에 하나씩** — 성공 전까지 다음으로 안 넘어감.
- **대기-성공(wait-for-success)** — 대상이 느리게 접근/정지, 실패해도 데미지 0, 프롬프트 반복.
- **강한 시각 신호** — 중앙 큰 텍스트 + 입력 힌트(PC 키캡 / 모바일 스와이프) + 대상 레인 하이라이트.

### 14.2 6단계 시퀀스 (`CONFIG.tutorial.steps`)
| # | 학습 | 성공 조건 | 힌트 |
|---|------|-----------|------|
| 1 | 자동 달리기 | 캐릭터 저절로 전진 시작 | "달리기는 저절로 돼요!" |
| 2 | 레인 이동 | 옆 레인 보석 1개 획득 | "← → / 좌우로 밀기" |
| 3 | 점프 | 느린 `LOW` 넘기 | "위로! 점프!" |
| 4 | 슬라이드 | 느린 `HIGH` 통과 | "아래로! 숙이기!" |
| 5 | 자동 사격 | 약한 몬스터(현재 월드 최약체) 처치 | "공격은 저절로!" |
| 6 | 스킬 안내 | 자동 ON: `AUTO` 배지 / OFF: Q·E 1회 | "스킬도 자동이에요" |

- 완료 후 → "준비 완료!" → `RUNNING_1`.

### 14.3 스킵 / 반복
- 우상단 **`건너뛰기`** 상시 → 즉시 `RUNNING_1`.
- **세션 플래그(sessionStorage `mhr_tutorial_seen`)** 로 "이미 봄" 기억 → 같은 세션 재시작 시 자동 스킵. 타이틀 `튜토리얼 다시 보기`로 재실행. 새 세션/새로고침이면 다시 노출.

## 15. 게임 흐름 / 상태머신

### 15.1 한 사이클 (월드마다)
```
타이틀(월드 선택) → (튜토리얼) → 스테이지 인트로 → 달리기1 →
[중간보스] → 달리기2(후반 난이도) → [최종보스] → 보상(장비+해금) →
결과(레벨·처치·획득·점수·다음 월드/재시작)
```

### 15.2 상태 정의 / 전환 (`GameStateName`)
| 상태 | 진입 동작 | 종료 → 다음 |
|------|-----------|-------------------|
| `TITLE` | 타이틀 UI, 월드 선택, 설정 | 시작 → 첫 진입 `TUTORIAL`, 이미 봤으면 `RUNNING_1` |
| `TUTORIAL`* | 안전 학습(무피해·무EXP) | 완료/건너뛰기 → `RUNNING_1` |
| `RUNNING_1` | 스테이지 인트로(3s, 탭 스킵) → 구간1 스폰·전투 | 목표 도달 → `MIDBOSS` / 체력0 → `GAMEOVER` |
| `MIDBOSS` | 아레나, 전진 정지, 보스 체력바 | HP0 → `RUNNING_2` / 체력0 → `GAMEOVER`(부활) |
| `RUNNING_2` | 구간2(후반 난이도), 전진 재개 | 목표 도달 → `FINALBOSS` / 체력0 → `GAMEOVER` |
| `FINALBOSS` | 아레나, 3페이즈 보스 | HP0 → `REWARD` / 체력0 → `GAMEOVER`(부활) |
| `REWARD` | 보상 장비 지급 + 외형 변화 + 다음 월드/스킬 해금 연출 | 탭 → `RESULT` |
| `RESULT` | 결과(레벨/처치/획득/점수/최고점수). 다음 월드/재시작/타이틀 | 선택 → `RUNNING_1`(다음 월드) / `TITLE` |
| `GAMEOVER` | 게임오버 UI, 점수. 부활 가능 시 버튼 | 부활 → 체크포인트 / 재시작 / `TITLE` |
| `PAUSE`(오버레이) | 시간 정지, 상태 보존, 자동 스킬 토글 | 계속 / 타이틀 |

\* `TUTORIAL`은 선택적. 세션 플래그 존재 또는 건너뛰기 시 직행.

### 15.3 월드 진행 / 해금
- 월드 N 클리어(REWARD) → `unlockWorld(N+1)`. `unlockedWorldIdx` = 클리어한 최대 월드 인덱스.
- 타이틀 월드 선택: 6버튼 그리드, 미해금은 🔒 잠금. 결과 화면 "다음 월드 ▶"로 직행 가능.
- 3월드 클리어 → 연사 폭주, 5월드 클리어 → 회복 파동 해금(§8).
- 6월드 클리어 → 결과 화면 "👑 몬스터 마스터 달성!".

### 15.4 체크포인트 / 부활
- **사망 시 보스 직전 체크포인트 부활**. 스냅샷: `playerState · inventory · stats · distance · runElapsed · gameState`.

### 15.5 공통 규칙 / 스폰 (`CONFIG.world`)
- `PAUSE`는 모든 플레이 상태에서 토글 가능(상태 보존, 시간 정지).
- 전환 시 입력 버퍼 초기화, 페이드/연출 허용.
- 구간 길이: **segment1 650 유닛 / segment2 850 유닛**.
- 스폰: `spawnAhead 45` 앞에서 생성, `despawnBehind 14` 뒤에서 제거.
- 몬스터 스폰 간격: 초반 4.0s → 후반 2.0s. 보스 아레나 거리 12.
- **동시 위협 상한 4**(장애물+몬스터 합).

## 16. UI / 화면 설계

- **타이틀**: 게임명, **월드별 최고점수**(선택 월드 기준 🏆), 월드 선택 6버튼(잠금 표시), 시작, 튜토리얼 다시 보기, 설정(자동 스킬·언어).
- **인게임 HUD (RUNNING)**: 체력바 / 점수·거리 / 동전·보석 카운트 / 레벨·EXP바 / 스킬 쿨다운(최대 4슬롯, 원형 게이지·AUTO 배지, 미해금 숨김).
- **보스 HUD**: 보스 이름 + 체력바(페이즈 눈금·색·잔상) + 플레이어 체력·스킬 + "약점!" 팝업.
- **스테이지 인트로**: RUNNING_1 진입 시 월드 이미지·이름 3초 표시(탭 스킵).
- **결과 화면**: 점수·최고점수(신기록 표시), 도달 레벨, 처치 수, 재화, 다음 월드/재시작/타이틀.
- **게임오버**: 도달 거리·레벨·재화·점수, (가능 시)체크포인트 부활, 재시작.
- **보상 화면**: 획득 장비 렌더 + (해당 시)스킬 해금 문구.
- 터치/마우스 공용. HUD는 노치/하단바 **세이프존 여백** 고려.
