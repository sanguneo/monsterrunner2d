# 🎮 몬스터 헌터 러너 — 최종 기획 및 설계서

> **버전**: 1.0 · **기준일**: 2026-06-10
> **개발 범위(이번 사이클)**: 1월드(학교 유령) **세로 슬라이스 MVP** 전체를 한 번에 구현.
> **수치 원칙**: 모든 밸런스 수치는 초기 표준값이며 `data/config` 단일 소스에서만 관리·조정한다.

## 0. 제품 한 줄 정의

> "달리고, 싸우고, 성장하며 6명의 몬스터 왕을 물리치는 **6~12세 어린이용 모바일 우선 2.5D 런닝 액션 RPG**"

- **타깃 사용자**: 6~12세 어린이 (막 기기를 다루는 연령 포함)
- **타깃 디바이스**: 중급폰 이상 (Galaxy S / iPhone 12+ 급), 세로(Portrait) 화면 우선
- **스택**: Three.js + TypeScript + Vite (브라우저 실행, 설치 불필요)
- **이번 마일스톤 목표**: 1월드 1스테이지를 `달리기 → 전투 → 중간보스 → 추가 달리기 → 최종보스 → 보상 → 결과`까지 **한 사이클로 완주 가능**하게 만들되, 성장·꾸미기·아이템 시스템을 **얕게라도 모두 연결**해 검증한다.

## 1. 전체 비전 (제품 로드맵)

플레이어는 몬스터로 가득한 세상을 달리며 장애물을 피하고, 몬스터를 처치하고, 아이템을 수집하며 성장한다. 6개 월드를 모두 클리어해 6명의 몬스터 왕을 물리치고 **👑 몬스터 마스터** 칭호를 얻는 것이 최종 목표다.

| 월드 | 테마 | 중간보스 | 최종보스 | 클리어 보상 |
|------|------|----------|----------|-------------|
| 🏫 1 | 학교 유령 | 유령 선생님 | 학교 괴담 소녀 | 유령 망토 |
| 🧟 2 | 좀비 마을 | 거대 좀비 | 좀비 킹 | 좀비 모자 |
| ⚡ 3 | 프랑켄슈타인 연구소 | 번개 골렘 | 프랑켄슈타인 | 번개 헬멧 |
| 🌊 4 | 바다마녀의 심해 | 크라켄 | 바다마녀 | 바다마녀 왕관 |
| 🦇 5 | 드라큘라 성 | 늑대인간 대장 | 드라큘라 | 드라큘라 망토 |
| 💀 6 | 해골왕의 성 | 해골 기사 | 해골왕 | 해골왕 왕관 |

**이번 사이클은 1월드만 구현**한다. 2~6월드 및 나머지 보스 10종은 다음 단계(§12 제외 범위)이며, 본 MVP의 아키텍처는 월드/몬스터/보스 추가가 데이터 확장만으로 가능하도록 설계한다.

### 핵심 재미 4축
1. **런닝 액션** — 자동 전진 + 점프 / 슬라이드 / 좌우 이동 / 타이밍 액션
2. **전투** — 자동 사격 + 수동 스킬, 보스 패턴 회피·약점 공격
3. **수집** — 동전 / 보석 / 회복 / 장비 / 꾸미기 아이템
4. **성장** — 레벨업 → 능력치 강화

## 3. 기술 / 아키텍처

### 3.1 렌더링·루프·좌표

- **렌더링**: Three.js (WebGL), 단일 페이지 풀스크린 캔버스.
- **게임 루프**: `requestAnimationFrame` 기반, **고정 타임스텝 업데이트** 권장(물리/충돌 안정).
- **좌표계**: 캐릭터는 +Z(또는 −Z) 방향 자동 전진. 레인은 X축 분할, 점프는 Y축.
- **카메라**: 캐릭터 뒤·위 오프셋 `(0, +4, −7)`, 부드러운 추적(lerp).
- **에셋 추상화**: 모든 시각 표현은 *시각(view)* 과 *로직(state)* 을 분리해 "프리미티브 → 모델" 교체 가능하게 한다.
  - 플레이어 = 캡슐/큐브 · 몬스터 = 색상 구체(종류별 색) · 장애물 = 박스/벽 · 아이템 = 회전 다면체 · 발사체 = 작은 구/실린더

### 3.2 모듈 구조 (TypeScript)

```

    Input.ts           // 키보드 + 터치 통합 입력 (입력 버퍼·코요테 타임 포함)
    Camera.ts          // 3인칭 추적
  entities/
    Player.ts
    Monster.ts
    Boss.ts
    Obstacle.ts
    Pickup.ts          // 동전/보석/회복

  systems/

    Combat.ts          // 자동 사격 + 스킬 + 충돌
    Progression.ts     // 경험치/레벨/능력치 자동 분배
    Inventory.ts       // 재화/아이템/장비
```

### 3.3 채택 아키텍처 패턴 (Seed `existing_patterns`)
- 상태머신 기반 게임 흐름
- 엔티티 view-logic 분리 (프리미티브 → 모델 교체)
- config 단일 소스 밸런싱 (수치는 `data/config`에서만)
- 패턴 큐 기반 보스 AI (예고 → 발동 → 후딜/경직 사이클)
- 패턴 템플릿 기반 장애물 스폰 (P1~P10, 안전 경로 보장)
- 거리 기반 난이도 램프 (진행도별 패턴 풀/간격 조절)
- 모듈 구조 (core / entities / systems / ui / data)

## 4. 좌표 / 레인 사양

- **레인 수**: 3 (0=좌, 1=중, 2=우)
- **레인 간격**: 2.0 유닛 → X 위치 = `(laneIndex − 1) * 2.0`
- **레인 이동**: 인접 레인 1칸씩, 보간 시간 ≈ 0.12s
- **시작 레인**: 중앙(1)

## 5. 러닝 액션 (코어 루프)

| 동작 | 키보드 | 터치 | 설명 |
|------|--------|------|------|
| 자동 전진 | — | — | 속도 자동, 시간/거리에 따라 점증 |
| 좌/우 이동 | ← / → | 좌/우 스와이프 또는 레인 터치 | 인접 레인 이동 |
| 점프 | ↑ / Space | 위 스와이프 | 공중 장애물·구덩이 회피 |
| 슬라이드 | ↓ | 아래 스와이프 | 낮은 장애물 통과 |
| 수동 스킬 | Q(슬롯1) / E(슬롯2) | 우하단 스킬 버튼 2개 | 쿨다운 기반 (§8) |

### 표준 수치
- 시작 전진 속도 **12** 유닛/s → 최대 **24** 유닛/s, 가속 **+0.5/s**(선형, 상한 고정)
- 점프: 초기 속도 상승 후 중력 **−25**, 공중 체공 ≈ **0.7s**
- 슬라이드: 지속 **0.6s**, 히트박스 높이 절반
- 충돌: 데미지 감소(§11 데미지 테이블) + 짧은 무적 **0.5s**. 구덩이 미점프 통과 시 큰 데미지.

## 6. 장애물 패턴 (러닝 구간)

스포너는 **패턴 템플릿**을 큐에서 뽑아 배치한다. 모든 템플릿은 **항상 통과 가능한 안전 경로**를 보장한다(어린이용 난이도 가드). 각 템플릿은 한 "비트(beat)" = 한 호흡 분량.

### 6.1 장애물 타입
| 타입 | 형태 | 회피 |
|------|------|------|
| `LOW` 낮은 장애물 | 지면 허들 | 점프 또는 레인 이동 |
| `HIGH` 높은 장애물 | 머리 위 가로막 | 슬라이드 또는 레인 이동 |
| `PIT` 구덩이 | 바닥 구멍 | 점프 |
| `BLOCK` 전체 벽 | 레인 일부 봉쇄 | 안전 레인 이동 (충돌 시 D 15 + 무적 0.5s) |

### 6.2 패턴 템플릿 (P1~P10)
| ID | 구성 | 해법 | 레인 점유 | 티어 |
|----|------|------|-----------|------|
| P1 단일허들 | 1레인 `LOW` | 점프 or 옆 레인 | 1/3 | 초반 |
| P2 가로막 | 1레인 `HIGH` | 슬라이드 or 옆 레인 | 1/3 | 초반 |
| P3 구덩이 | 1레인 `PIT` | 점프 | 1/3 | 초반 |
| P4 2레인 차단 | 2레인 `BLOCK`(안전 1레인) | 안전 레인 이동 | 2/3 | 중반 |
| P5 전열 허들 | 3레인 `LOW` | 전체 점프 | 3/3 | 중반 |
| P6 전열 가로막 | 3레인 `HIGH` | 전체 슬라이드 | 3/3 | 중반 |
| P7 슬라럼 | `BLOCK` 좌→중→우 순차(3비트) | 연속 레인 이동 | 1/3씩 | 중후반 |
| P8 점프-슬라이드 콤보 | `LOW` 후 즉시 `HIGH`(같은 레인) | 점프 후 슬라이드 | 1/3 | 후반 |
| P9 보상 아치 | `LOW` 위에 동전 5개 호 | 점프(회피+수집 동시) | 1/3 | 전 구간 |
| P10 몬스터+장애물 | `BLOCK` 1레인 + 인접 레인 몬스터 | 이동 후 사격 | 2/3 | 후반 |

### 6.3 난이도 램프 (진행도별 출현 풀)
| 진행도 | 출현 패턴 | 패턴 간격 |
|--------|-----------|-----------|
| 0~30% (초반) | P1, P2, P3, P9 | 2.5s |
| 30~70% (중반) | P1~P7, P9 | 1.8s |
| 70~100% (후반) | P4~P10 | 1.2s |

### 6.4 배치 규칙
- 연속 동일 템플릿 **2회 초과 금지**(단조로움 방지).
- `HIGH` 직후 `PIT` 등 **물리적 동시해법 불가 조합**은 같은 레인 0.8s 이내 금지.
- 점프(≈0.7s)/슬라이드(0.6s) 동작 시간을 고려해 다음 장애물까지 **최소 회복 간격 0.8s** 확보.
- **동시 위협 상한 4** (몬스터 스폰과 합산 관리).
- P9(보상 아치)는 난이도 휴식 + 동전 유도용으로 전 구간 주기 삽입.

## 7. 전투 사양

### 7.1 자동 사격
- 사정거리 **18 유닛** 내 **가장 가까운 몬스터** 자동 타겟.
- 발사 간격 **0.4s 고정**(초당 2.5발). 성장 능력치 비대상이며, 스킬 '연사 폭주'만 일시 단축.
- 발사체 데미지 = 플레이어 공격력 (치명타 시 ×2).
- 처치 시 경험치 + 동전/보석 드랍.
- ⚠️ 플레이어 피격 히트박스는 80% 축소되지만, **자동 사격 명중 판정은 정상 크기 유지**(§13.2).

### 7.2 1월드 일반 몬스터 (임시 도형)
| 몬스터 | 색(임시) | 체력 | 접촉 데미지 | 행동 | 처치 EXP |
|--------|----------|------|-------------|------|----------|
| 책 유령 | 갈색 구체 `0x8b5a2b` | 20 | 10 | 제자리/느린 접근 | 8 |
| 연필 유령 | 노랑 구체 `0xffd54a` | 15 | 10 | 직선 전진 | 6 |
| 종이비행기 귀신 | 흰 구체 `0xffffff` | 12 | 8 | 좌우 흔들며 접근 | 5 |

## 8. 수동 스킬 시스템

> **스킬 풀 4종 정의, MVP 기본 장착 2종.** 슬롯제로 만들어 향후 해금/교체 확장 여지를 둔다.

### 8.1 스킬 풀
| 슬롯 | 스킬 | 효과 | 쿨다운 | MVP |
|------|------|------|--------|-----|
| 1 | **광역 폭발** | 전방 범위 몬스터·보스에 `공격력 ×4` 즉시 피해 | 8s | ✅ 장착 |
| 2 | **무적 대시** | 1.5s 무적 + 전진속도 +50% + 장애물·몬스터 관통 | 12s | ✅ 장착 |
| 3 | **연사 폭주** | 5s간 사격 간격 −50%(연사 2배) | 14s | 해금 예정(로직만) |
| 4 | **회복 파동** | 즉시 체력 +40 + 주변 잡몹 약한 넉백 | 20s | 해금 예정(로직만) |

### 8.2 슬롯 / 입력 / 자동 발동
- MVP는 **슬롯 2개 고정**(폭발=1, 대시=2). 입력: 키보드 `Q`/`E`, 터치 우하단 버튼 2개.
- HUD에 각 슬롯 **쿨다운 원형 게이지** 표시. 쿨다운 중 입력 무시(시각적 비활성).
- **자동 발동 토글(기본 ON)** — §13.1 참조.
- 스킬 3·4는 데이터·로직만 준비하고 UI 비노출.

### 8.3 설계 의도
- 광역 폭발 = 잡몹 정리 + 보스 경직창 폭딜 ("공격 강화")
- 무적 대시 = 회피 불가 상황 탈출 + 어린이 실수 보정 ("생존")

## 9. 보스전 사양

**진입**: 일정 거리 도달 → 전용 보스 아레나(전진 정지, 카메라 보스 정면). 보스 체력바 + 이름 표시.
**구조**: 보스는 *패턴 큐*를 순환 실행. 각 패턴 = `예고(telegraph) → 발동(active) → 후딜(recovery)`.
**경직(stagger)**: 일부 패턴 종료 후 무방비. 이 창 동안 **받는 데미지 ×1.5**(★약점).

### 9.1 중간보스 — 유령 선생님 (HP 400, 2페이즈)
| # | 패턴 | 예고 | 발동 | 회피 | D | 후딜/경직 |
|---|------|------|------|------|---|-----------|
| A | 분필 던지기 | 0.6s (대상 레인 바닥 마커) | 1레인 분필 투사체(속도 20) | 해당 레인 벗어나기 | 12 | 후딜 0.4s |
| B | 칠판 충격파 | 0.8s (칠판 들기 + 전 레인 바닥 발광) | 지면 충격파 전 레인 통과 | **점프** | 18 | **경직 1.2s** ★ |
| C | 유령 소환 | 1.0s (소환진) | 잡몹 유령 1~2기 소환(접근형) | 처치/회피 | 접촉 8 | 후딜 0.6s |

- **Phase 1 (100%→50%)**: 큐 `A → C → B`, 간격 1.0s. 칠판 충격파 후 1.2s 경직이 주 딜 타이밍.
- **Phase 2 (50%→0%, 가속)**: 예고 −25%, 간격 0.7s. 분필 **2레인 동시**(안전 1레인), 경직 1.2s→0.9s. 큐 `A2 → B → A2 → C`.
- **예상 전투 시간**: L4(43 DPS) 기준 순수 피격 ≈ 9.3s, 회피 포함 실전 **25~35s**.

### 9.2 최종보스 — 학교 괴담 소녀 (HP 700, 3페이즈)
| # | 패턴 | 예고 | 발동 | 회피 | D | 후딜/경직 |
|---|------|------|------|------|---|-----------|
| T | 순간이동 | 사라짐 0.5s → 재등장 0.4s | 좌/중/우 새 위치 이동 후 즉시 연계 | (연계 패턴 따름) | — | — |
| H | 머리카락 공격 | 0.7s (대상 레인 그림자) | 1~2레인 머리카락 벽 2.0s 봉쇄 | 안전 레인 이동 | 접촉 20 | 후딜 0.5s |
| S | 공포의 비명 | 1.0s (귀 막기 + 화면 진동) | 전 레인 광역 음파(1.0s) 동안 **자동사격 봉인** | **슬라이드** 회피(성공 시 D 0) | 15 | active 후 **경직 1.5s** ★ |

> **비명 타이밍 규칙(겹침 방지)**: `예고 1.0s → active 1.0s(이 동안만 자동사격 봉인) → 경직 1.5s(사격 복구·×1.5)`. 봉인과 경직은 **순차**이며 절대 겹치지 않는다. 수동 스킬은 봉인 중에도 사용 가능 → 봉인 구간 카운터플레이 제공.

- **Phase 1 (100%→66%)**: 큐 `H → T → H`, 간격 1.2s. 머리카락 1레인.
- **Phase 2 (66%→33%)**: 큐 `T → H → S → H`, 간격 1.0s. 머리카락 2레인(안전 1). 비명 경직 1.5s가 주 딜.
- **Phase 3 (33%→0%, 광폭화)**: 예고 −30%, 간격 0.6s. 콤보 `순간이동 → 머리카락(2레인) → 비명`, 경직 1.5s→1.0s. 큐 `T+H → S → T+H`.
- **승리**: HP 0 → 유령 망토 + 재화 지급 → 결과 화면.
- **예상 전투 시간**: L5(49 DPS) 기준 순수 피격 ≈ 14.3s, 회피 포함 실전 **50~65s**.

### 9.3 보스 공통 규칙
- 예고 동안 회피 입력 가능(레인 이동/점프/슬라이드).
- 경직(★) 창에 사격·스킬을 몰아넣는 것이 핵심 딜링 설계.
- 페이즈 전환 시 짧은 무적 0.5s + 패턴 큐 리셋.
- 보스 접촉/투사체 피격 시 플레이어 무적 0.5s.

### 9.4 약점 / 경직 연출 (피드백 디자인)
**경직(★약점) 진입**: 보스 외곽 **빨강 윤곽 발광** + 본체 빨강 펄스 / 머리 위 **"약점!" 팝업** + 화살표 깜빡임 / 받는 데미지 ×1.5 동안 **데미지 숫자 확대·노란색** / 타격 스파크 강화. (선택) 0.2s 미세 슬로우는 폴리싱 단계.

**예고 신호 표준** (회피 학습용 일관 신호):
| 패턴 | 예고 신호 |
|------|-----------|
| 분필 던지기 | 대상 레인 바닥 빨강 마커 + 보스 팔 들기 |
| 칠판 충격파 | 전 레인 바닥 발광 라인 + 칠판 들어올림 |
| 유령 소환 | 소환진 원 확대 |
| 머리카락 공격 | 대상 레인 그림자 짙어짐 |
| 공포의 비명 | 귀 막기 모션 + 화면 진동 + 가장자리 어둡게 |

**페이즈 전환**: 무적 0.5s 동안 화면 짧은 플래시 + 포효 모션 + 체력바 색 변경(P1 초록 → P2 주황 → P3 빨강). 소녀 P3 진입 시 배경/조명 톤 더 어둡게.
**체력바**: 피해 시 흰색 잔상(딜레이 감소) / 페이즈 경계(50% · 66%·33%)에 눈금 표시.

## 10. 성장 시스템

```
몬스터 처치 / 보석 수집 → 경험치 → 레벨업 → 능력치 자동 분배
```

### 10.1 능력치
| 능력치 | 초기값 | 레벨업 효과 |
|--------|--------|-------------|
| 체력 | 100 | +10/레벨 |
| 공격력 | 10 | +2/레벨 |
| 이동속도 | (전진속도 영향) | +2%/레벨 (전진속도 상한도 같이 상향) |
| 점프력 | 기본 | +3%/레벨 |
| 치명타 확률 | 5% | +1%/레벨 (배수 ×2) |

- 경험치 곡선: **필요 EXP = `50 * level`** (선형 누적).
- 레벨업은 **자동 분배**(MVP 확정값).

### 10.2 DPS / 성장 곡선 검산
- 유효 DPS = `공격력 × 2.5 × (1 + 치명타확률)`, 최대 체력 = `100 + 10*(level−1)`

| 레벨 | 공격력 | 치명타 | 유효 DPS | 최대 체력 |
|------|--------|--------|----------|-----------|
| 1 | 10 | 5% | 26.3 | 100 |
| 3 | 14 | 7% | 37.5 | 120 |
| 5 | 18 | 9% | 49.1 | 140 |
| 8 | 24 | 12% | 67.2 | 170 |

### 10.3 EXP 경제 & 예상 도달 레벨
| EXP 획득원 | EXP | | 획득원 | EXP |
|------------|-----|---|--------|-----|
| 책 유령 처치 | 8 | | 보석 수집 | 10 |
| 연필 유령 처치 | 6 | | 중간보스 처치 | 120 |
| 종이비행기 처치 | 5 | | 최종보스 처치 | 250 |

- **현실 엔딩 레벨 ≈ L5~6**: 1구간 종료 ~L3 → 중간보스 후 ~L4 → 2구간 후 ~L4~5 → 최종보스 직전 ~L5 → 처치 후 ~L5~6.
- ⚠️ **튜닝 레버**: 엔딩을 L7~8로 올리려면 몬스터/보석 EXP를 1.5~2배로 상향하거나 곡선을 `40*level`로 완만화. 보스 HP(400/700)는 현실 도달 레벨(L4/L5)에 맞춰 역산됨 — 곡선 변경 시 보스 HP 재검산 필요.

## 13. 접근성 / 조작 보조 (어린이 진입 문턱 완화)

> 채택 범위: **① 자동 스킬 토글 + ② 입력 관용**만. (PC 키 클러스터링·탭존·키즈 쉬움 모드·터치 타깃 확대는 이번 MVP 미적용.)

### 13.1 자동 스킬 발동 (토글, 기본 ON)
- 장착 스킬(슬롯1·2)이 **쿨다운 종료 즉시 자동 발동**.
  - 광역 폭발: 전방 사정거리 내 몬스터/보스 있을 때 자동(헛방 방지).
  - 무적 대시: 전방 회피 불가 위협 감지 시 우선 발동, 없으면 쿨마다.
- **OFF 전환** 시 수동 입력(Q/E·터치)으로 발동.
- 토글 위치: 타이틀 설정 + 일시정지 메뉴, 게임 중 즉시 전환.
- HUD: 자동 모드일 때 슬롯에 **`AUTO` 배지** 표시.

### 13.2 입력 관용 (Forgiveness)
| 항목 | 값(표준) | 설명 |
|------|----------|------|
| 입력 버퍼 | 0.15s | 점프/슬라이드/레인이동을 조금 미리 눌러도 가능 타이밍에 자동 발동 |
| 코요테 타임 | 0.12s | 착지 직전/직후 입력도 점프로 인정 |
| 히트박스 축소 | 시각의 80% | 플레이어 **피격 판정에만** 적용(자동 사격 명중 판정은 정상 크기) |
| 액션 중 입력 큐 | 1개 | 동작 중 다음 입력 1개 큐잉, 종료 즉시 실행 |

## 14. 튜토리얼 / 첫 실행 온보딩

> 목표: 막 기기를 다루는 어린이가 핵심 액션을 *실패해도 처벌 없이* 하나씩 배운다.
> 형태: `RUNNING_1` 시작 직전의 **안전 학습 구간**(무피해·무게임오버·무EXP).

### 14.1 진행 방식
- **한 번에 하나씩** — 해당 동작 성공 전까지 다음으로 안 넘어감.
- **대기-성공(wait-for-success)** — 대상이 느리게 다가오거나 잠시 정지, 성공까지 대기. 실패해도 데미지 0, 프롬프트 반복.
- **강한 시각 신호** — 중앙 큰 텍스트 + 입력 힌트 아이콘(PC 키캡 / 모바일 스와이프 손동작) + 대상 레인 하이라이트.
- **자동 요소 안내** — 사격·(자동 시)스킬은 "직접 안 눌러도 돼요".

### 14.2 6단계 시퀀스

| # | 학습 | 성공 조건 | 힌트 |
|---|------|-----------|------|
| 1 | 자동 달리기 | 캐릭터 저절로 전진 시작 | "달리기는 저절로 돼요!" |
| 2 | 레인 이동 | 옆 레인 보석 1개 획득 | "← → / 좌우로 밀기" + 레인 하이라이트 |
| 3 | 점프 | 느린 `LOW` 허들 1개 넘기 | "위로! 점프!" + ↑ / 위 스와이프 |
| 4 | 슬라이드 | 느린 `HIGH` 가로막 1개 통과 | "아래로! 숙이기!" + ↓ / 아래 스와이프 |
| 5 | 자동 사격 | 약한 몬스터 1기 자동 처치 | "공격은 저절로!" |
| 6 | 스킬 안내 | 자동 ON: `AUTO` 배지 강조 / OFF: Q·E 1회 시전 | "스킬도 자동이에요" or "Q/E로 필살기!" |

- 완료 후 → "준비 완료!" 연출 → `RUNNING_1` 본편 시작.

### 14.3 스킵 / 반복
- 우상단 **`건너뛰기`** 상시 노출 → 즉시 `RUNNING_1`.
- **세션 플래그(sessionStorage `mhr_tutorial_seen`)** 로 "이미 봄" 기억 → 같은 세션 재시작 시 자동 스킵(타이틀 `튜토리얼 다시 보기`로 재실행).
- 새 세션/새로고침이면 다시 노출.

## 15. 게임 흐름 / 상태머신

### 15.1 한 사이클
```
타이틀 → (튜토리얼) → 달리기1 + 장애물·몬스터·아이템 → [중간보스: 유령 선생님] →
달리기2(후반 난이도) → [최종보스: 학교 괴담 소녀] → 보상(유령 망토 + 재화) →
결과 화면(레벨·처치·획득·재시작)
```

### 15.2 상태 정의 / 전환
| 상태 | 진입 동작 | 종료(전환) → 다음 |
|------|-----------|-------------------|
| `TITLE` | 타이틀 UI, 입력 대기 | 시작 → 첫 진입이면 `TUTORIAL`, 이미 봤으면 `RUNNING_1` |
| `TUTORIAL`* | 안전 학습(무피해·무게임오버·무EXP) | 6단계 완료/건너뛰기 → `RUNNING_1` |
| `RUNNING_1` | 구간1 스폰, 전진/전투 활성 | 구간1 목표 도달 → `MIDBOSS` / 체력0 → `GAMEOVER` |
| `MIDBOSS` | 아레나 진입, 전진 정지, 보스 체력바 | HP0 → `RUNNING_2` / 체력0 → `GAMEOVER`(체크포인트 부활) |
| `RUNNING_2` | 구간2 스폰(후반 난이도), 전진 재개 | 구간2 목표 도달 → `FINALBOSS` / 체력0 → `GAMEOVER` |
| `FINALBOSS` | 아레나, 3페이즈 보스 | HP0 → `REWARD` / 체력0 → `GAMEOVER`(체크포인트 부활) |
| `REWARD` | 유령 망토 + 재화 지급 연출 | 연출 종료 → `RESULT` |
| `RESULT` | 결과(레벨/처치/획득/재시작) | 재시작 → `TITLE`(또는 `RUNNING_1`) |
| `GAMEOVER` | 게임오버 UI, 점수 표시 | 확인 → `TITLE` |

\* `TUTORIAL`은 선택적 상태. 세션 플래그 존재 또는 건너뛰기 시 직행.

### 15.3 체크포인트 / 부활
- **사망 시 보스 직전 체크포인트에서 부활**. 체크포인트는 보스 진입 직전의 `playerState · distance · gameState` 스냅샷.

### 15.4 공통 규칙
- `PAUSE`는 모든 플레이 상태에서 토글 가능한 오버레이(상태 보존, 시간 정지).
- 전환 시 입력 버퍼 초기화, 0.3s 페이드/연출 허용.
- 상태별 업데이트 책임: `RUNNING*` = Spawner+Combat+Progression / `BOSS` = Boss 패턴큐+Combat.

### 15.5 스폰 / 난이도 램프 (러닝 구간)
- 구간 길이: 시간 기준 약 40~60s (전진속도 가속 반영, 거리 환산).
- 장애물 밀도: 초반 2.5s당 1개 → 후반 1.2s당 1개.
- 몬스터 스폰: 초반 4s당 1기 → 후반 2s당 1~2기.
- **동시 위협 상한 4** (장애물+몬스터 합).

## 16. UI / 화면 설계

- **타이틀**: 게임명, 시작 버튼, 설정(자동 스킬 토글·언어), 튜토리얼 다시 보기.
- **인게임 HUD (RUNNING)**: 체력바 / 점수·거리 / 동전·보석 카운트 / 레벨·EXP바 / 스킬 쿨다운 2종(원형 게이지·AUTO 배지).
- **보스 HUD**: 보스 이름 + 체력바(페이즈 눈금) + 플레이어 체력·스킬.
